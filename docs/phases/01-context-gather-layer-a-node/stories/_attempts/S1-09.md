### Design-pattern lens — refactor checklist walked

- **Functional core / imperative shell:** `raw_truncation.py` is pure
  (purity test enforces); `cli.py` owns I/O + structlog emission. ✓
- **Tagged union (sum type):** `TruncationOutcome = Untruncated | Truncated`;
  the CLI dispatch is one `isinstance(outcome, Truncated)`. Illegal states
  unrepresentable — `Truncated` requires both `original_bytes` and
  `budget_bytes`; `Untruncated` carries nothing. ✓
- **Open/Closed at the file boundary:** `ResourceBudget` is the kernel
  (typed knob surface); per-probe overrides are policy (data, attached via
  the class attribute `declared_resource_budget`). Adding a future
  resource dimension is one more field; no probe edit, no coordinator
  edit. ✓
- **Smart-constructor invariant:** `ResourceBudget.__post_init__` rejects
  `truncate_mb > raw_artifact_mb` at construction (Rule 12 — fail loud).
  Equality at the limit allowed (matches `>` boundary semantics). ✓
- **Chokepoint preserved:** `Writer.write` signature/body untouched
  (ADR-0011). Truncation lives upstream of the chokepoint. ✓
- **Newtype/registry/strategy:** not warranted at this surface size;
  premature abstraction would violate Rule 2.

### Anti-patterns — none introduced

No primitive-obsession (the outcome is a tagged union); no anaemic types
(both `Truncated` and `ResourceBudget` carry their invariants); no
hidden state (pure helper); no pure/impure tangle (the CLI is the sole
side-effect site).
