### Defects fixed during attempt

#### D-1 — docstring substring missed by validator-pinned test

The validator-pinned `test_module_docstring_names_arch_adr_and_msgpack_rejection`
asserts `"ADR-0002" in doc`. The first-draft docstring named the ADR by
filename only (`0002-parsed-manifest-memo-on-probe-context.md`) but not the
short identifier "ADR-0002". Resolution: kept the filename reference and
prefixed it with `ADR-0002 —`. The substring contract is what matters; the
filename is wrapping (L-3 / L-7 family — story prose is the *intent*; the
substring is the *contract*).

#### D-2 — bound-method `id()` is unstable; cross-/same-gather tests need `__self__`

The hardened TDD plan prescribed `seen.append(id(ctx.parsed_manifest))` for
AC-17 and AC-18. Python builds a *fresh* bound-method object on every
attribute access of `memo.get`, so `id(ctx.parsed_manifest)` is not stable
across distinct ctx instances even when the underlying memo is the same
instance — and identical across two memos when the first bound-method is
GC'd and the slot is recycled. The load-bearing invariant is the
*underlying memo identity*, not the bound-method identity. Resolution:
capture `ctx.parsed_manifest.__self__` (the underlying memo instance)
instead and compare with `is` / `is not`. Both AC contracts hold:

- AC-17 (cross-gather): `seen[0] is not seen[1]` — two memos, distinct.
- AC-18 (same-gather): `seen[0] is seen[1]` — same memo, shared.

Surfaced as L-13 in `_lessons.md` so the next executor doesn't re-litigate.
(Sibling to L-9 — when the story's literal identity assertion disagrees
with Python's runtime model, preserve the behavioral contract; widen the
identity check.)

#### D-3 — cache hit short-circuits the second `gather()` in AC-17

`FakeProbe.cache_key` returns `f"sha256:{self.name}"` — same name across
two gathers ⇒ same cache key ⇒ second gather hits the cache and short-
circuits before the probe `run()` body executes ⇒ `seen.append(...)` runs
only once. The fix was to name the two probes distinctly
(`stub-a-1` / `stub-a-2`) so each gather goes through the dispatch path.
The invariant under test is memo-instance identity per gather, not cache
behavior — distinct names are inert to that invariant. Documented inline
in the test body so a future reader doesn't think the renaming is
load-bearing for the AC.

### Refactor decisions (design-pattern lens)

Walked the refactor lens at GREEN time:

- **Kernel/policy split / Open/Closed.** Already done (allowlist as
  constructor kwarg). The shape the validator pinned. No further work.
- **Capability pattern.** `parsed_manifest=memo.get` is the canonical
  capability injection — probes receive a callable, not the memo type.
- **Dependency inversion.** `BudgetingContext.parsed_manifest` is typed as
  `Callable[[Path], Mapping[str, Any] | None] | None` — the consumer does
  not depend on `ParsedManifestMemo`'s concrete type. Phase 2's wider
  allowlist is a constructor-arg change in the coordinator, not an edit to
  this module.
- **Newtype on cache key `(str, int, int)`.** Currently a bare tuple.
  Deferred — S1-08 flips the key to `(content_hash,)` sourced from
  `ctx.input_snapshot`, at which point a `_CacheKey` smart constructor
  earns its keep. Today the tuple's lifetime is one class instance and
  the cost outweighs the benefit.
- **Type-alias for the `Callable[[Path], Mapping[str, Any] | None]`
  signature.** Duplicated between `budget.py` and `coordinator.py` (two
  call sites). Not lifted — two duplicates is not yet rule-of-three; the
  alias would obscure the shape on read.
- **No anti-patterns introduced.** No primitive obsession beyond
  `path: str` in the cache key (S1-08 will replace). No hidden state
  (memo is per-gather, owned by `gather()`). No pure-impure tangle
  (the memo is a pure-on-input function with a single internal dict
  insert; failure does not cache). No anaemic types (the memo is a class
  for state, not a data-bag).
