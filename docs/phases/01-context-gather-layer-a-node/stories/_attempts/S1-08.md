### What happened

Single attempt; all 23 ACs landed clean. Three runtime surprises required mid-attempt adjustments — all caught by the test suite (not by manual review), all fixed without rewriting the implementation:

#### Surprise 1 — Symlink path canonicalization had a buried inconsistency between AC-10 + T-6.

AC-10 prescribes `InputFingerprint(path=str(matched_path.resolve()), ...)` for the refused-symlink branch. But `matched.resolve()` on a symlink follows it — for `tmp/package.json → tmp/real.json`, the fingerprint's `path` ends in `real.json`. T-6 asserts `fp.path.endswith("package.json")` (the symlink's own name, not the target's).

Resolution: factored out `_canonical_abs_path(matched) = str(matched.parent.resolve() / matched.name)` — canonicalizes parent components without following the final-component symlink. For regular files this is identical to `str(matched.resolve())` (T-8 still pins the regular-file case); for symlinks the link's own name is preserved (T-6 passes). The protocol surfaces both halves of the AC-12 roundtrip without contradicting AC-10.

#### Surprise 2 — `ctx.parsed_manifest.__self__` accessor broke for S1-07's coordinator-injects-memo tests.

S1-07's `test_cross_gather_isolation` + `test_same_gather_sharing_across_probes` capture `ctx.parsed_manifest.__self__` to identify the underlying memo (L-13 cleanup). S1-08 wires `ctx.parsed_manifest = make_parsed_manifest_adapter(snapshot, memo)` — a closure, no `__self__`.

Resolution: expose the memo on the closure as `_adapter.__memo__ = memo` and replace `__self__` with `__memo__` in the four test sites in `test_coordinator_injects_memo.py`. The S1-07 test contract (instance-identity invariant for same-gather sharing / cross-gather isolation) is preserved; only the accessor name changed. S1-07's `test_parsed_manifest_memo.py` (the memo's own contract tests) stayed untouched as the story prescribed.

This is a recurring pattern — see L-15 below.

#### Surprise 3 — `probe.input_snapshot.*` events missed the ambient `run_id`.

`tests/unit/test_coordinator.py::test_every_lifecycle_event_carries_run_id` asserts every `probe.*` event carries `run_id` explicitly. `structlog.testing.capture_logs` skips `merge_contextvars`, so contextvars-bound `run_id` does NOT auto-attach to log events under test. The CLI module's comment at `src/codegenie/cli.py:579` calls this out: "Bind contextvars so child events inherit `run_id`; ALSO pass it explicitly so test-only processor chains (...) still see it."

Resolution: in `compute_input_snapshot`, read `run_id = structlog.contextvars.get_contextvars().get("run_id")` at the top of the function and pass `run_id=run_id` explicitly to all three emitted events. The pattern matches `coordinator.py:_dispatch_one` exactly (line ~291). See L-16 below — this is a contract every new event-emitting site in this codebase must follow.
