### Bugs uncovered by the new tests (fixed in this story)

1. **`_parse_handrolled` rejected zero-entry bodies.** The empty corpus
   fixture (`empty/yarn.lock` ‚Äî comments-only) raised
   `ValueError("no yarn.lock entries parsed")` which `parse()`
   translated to `MalformedLockfileError`. Per story step 3, fixed in
   `src/codegenie/probes/_lockfiles/_yarn.py`: removed the
   `if not entries: raise` line; comments-only bodies now return
   `{"entries": {}}`. Malformed bytes (non-comment, non-header lines)
   still raise via the "expected entry header" branch. Existing S3-03
   tests still green (no test relied on the raise ‚Äî the docstring at
   `test_yarn.py:93` is mutation-caught-narrative, not a `pytest.raises`
   assertion).

2. **`_parse_handrolled` mishandled comma-joined quoted headers.** The
   parity test caught a real divergence on the
   `multi_spec_shared_header` fixture:
   - `pyarn` emits entry-key `'debug@^2.6.9, debug@^2.6.0'`
   - hand-rolled emitted `'debug@^2.6.9", "debug@^2.6.0'` (only outer
     quotes stripped; embedded `", "` preserved).

   Extracted a `_dequote_entry_header(header)` helper that splits a
   quoted multi-spec header on `'", "'` and re-joins with `', '`.
   Single-spec headers (`lodash@^4.17.21`) and quoted single-spec
   headers (`"@types/node@^20"`) round-trip correctly. All 18 existing
   `test_yarn.py` cases still green; all 4 parity cases green; all 8
   oracle cases green.

### Mypy override (pyarn install-state)

S3-03's `# type: ignore[import-not-found]` on `import pyarn.lockfile`
broke locally once pyarn was installed for the parity-test sanity check
(mypy reports `import-untyped` instead ‚Äî pyarn ships no `py.typed`).
Replaced the per-line ignore with a `pyproject.toml` module-level
override (`module = ["pyarn", "pyarn.*"]`, `ignore_missing_imports =
true`) so the same source compiles cleanly under both install states.
The override is the natural home for an optional dependency ‚Äî the
per-line ignore would flip-flop on every install-state change.

### Refactor decisions (design-pattern lens)

- **Functional core / imperative shell** ‚Äî `_check_invariant_*` helpers
  are pure `(YarnLock, str) -> None`; `_dequote_entry_header` is pure
  `str -> str`. The only imperative shell is the `_yarn.parse(path)`
  call site, which already lives behind `open_capped`.
- **Extension by addition** ‚Äî adding a fourth invariant is `def
  _check_invariant_4` + `def test_invariant_4_catches_*` + a new
  mutator; zero edits to existing tests or the parser. Adding a third
  parser strategy would mean extracting a `_check_invariants` registry
  (rule of three, not yet hit per the validator's Notes ¬ß10).
- **Smart constructor (deferred).** `_dequote_entry_header` is a thin
  normalizer, not a smart constructor ‚Äî `YarnLock` keys remain raw
  strings, and `NodeManifestProbe` (S3-05) is the right place to wrap
  them in a typed `EntrySpec` newtype if and when the codebase needs
  one. Recorded per Rule 2 (no premature abstraction).
- **Rule of two on the invariant helpers** ‚Äî `_check_invariant_*` live
  in the oracle module and are imported by the self-check module.
  Two consumers ‚Üí extraction stays inline. A third consumer (e.g., a
  yarn-berry-only oracle in a future phase) is the right moment to
  extract to `_oracle_helpers.py`.

### Lessons promoted to `_lessons.md`

- L-4 candidate: a property-based oracle is worth at least one
  parser-bug callout per story it lands in. The empty fixture caught
  one bug; the parity test caught another. The bytes-vs-implementation
  contract is the load-bearing reason both fired.
- L-5 candidate: optional dependencies (pyarn) benefit from
  pyproject-level `ignore_missing_imports` rather than per-line ignores
  ‚Äî the per-line variant flips on every install-state change and
  forces a `# type: ignore[import-untyped]` ‚Üî `# type:
  ignore[import-not-found]` swap.
