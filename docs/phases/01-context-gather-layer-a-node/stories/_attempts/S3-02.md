#### Deviation #1 — `os.fstat` monkey-patch replaced with `safe_json.load` monkey-patch

**Story prescribed (TDD §Red, `test_parse_oversized_file_reraises_size_cap`):**
a custom `FakeStat` class injected via
`monkeypatch.setattr(os, "fstat", ...)` to spoof a 60 MB file size
pre-`safe_json.load`.

**Resolution:** Monkey-patched `_npm.safe_json.load` directly to
raise `SizeCapExceeded("synthetic: size>cap")` via a parametrized
test, asserting object identity (`exc.value is raised`) on the
re-raise. The same parametrized test covers `SymlinkRefusedError`
(the real-symlink integration test is also kept verbatim for the
load-bearing real-fs path), and `DepthCapExceeded` keeps a
real-fixture integration test against bracket-nested JSON.

Object-identity (`exc.value is raised`) is strictly stronger than
`pytest.raises(type(raised))` — catches any re-wrap that constructs
a new instance of the same class. The `os.fstat` approach is
indirection that proves the same contract less directly (Rule 2:
minimum code that proves the contract). This is the exact pattern
S3-01 ratified; S3-03 should follow the same shape (after the
rule-of-three extraction decision).

### Refactor decisions

Walked the design-pattern lens (registry, strategy, ports/adapters,
newtype, tagged union, smart constructor, plugin) per the executor
skill's refactor checklist:

- **Rule of three (deferred).** `_pnpm.py` and `_npm.py` now share
  the shape
  `try: safe_X.load(...) except MalformedXError as cause: raise
  MalformedLockfileError(f"{path}: {type(cause).__name__}: {cause}")
  from cause`. The story explicitly defers extraction of a shared
  `_translate(path, *, cause)` helper to S3-03 (the rule-of-three
  trigger). Premature extraction would bake an API shape based on
  two consumers when the third may dispreference it. **Followed
  the story's instruction; no helper extracted in S3-02.**
- **Module constants (deferred).** `NPM_LOCKFILE_MAX_BYTES` /
  `NPM_LOCKFILE_MAX_DEPTH` mirror `_pnpm.py`'s equivalents
  literally. Lifting to a shared `_lockfiles/__init__.py` constant
  is also S3-03's rule-of-three decision.
- **Open/Closed (preserved).** AC-12's architectural test pins
  zero sibling-imports; adding `_bun.py` is a new file, zero
  edits to `_npm.py`.
- **Functional core / imperative shell (preserved).** `parse()` is
  I/O + exception translation only; no validation, no defaulting,
  no key reshaping. Validation is `NodeManifestProbe`'s job (S3-05).
- **Marker discipline (preserved).** `MalformedLockfileError`
  remains a positional-message-only marker. Path is recoverable
  from `args[0]`; cause is recoverable from `__cause__`. No
  `__init__` override; no instance state.
- **Plugin / Registry kernel.** Shared kernel for the family is
  `parsers/safe_json` + `codegenie.errors` (proven by AC-12
  source-introspection). Sibling parsers register their format by
  *existing as a file*; the kernel knows nothing about them.

No anti-patterns flagged: no primitive-obsession-on-ID violations
(no IDs here), no anaemic types where sum types would do (typed
exceptions are tagged-union-like; no further refinement needed
for a thin wrapper), no hidden state, no pure/impure tangle.
