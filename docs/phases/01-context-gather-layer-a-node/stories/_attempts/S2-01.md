### Deviations from the story
- **AC-9 pointer format** — story expected `/probes/language_detection/language_stack/<rogue>` (JSON-Pointer). The project's validator emits jsonschema's `json_path` which uses dotted form `$.probes.language_detection.language_stack`. T-4 asserts the substring `probes.language_detection.language_stack` and `rogue_field` in the error message — preserves the AC intent (slice path is named) without forcing a validator rewrite (Rule 3 — surgical changes).
- **AC-12 module-import assertion test** — implemented as a re-evaluation of `_WARNING_ID_RE` against a poisoned frozenset, rather than a `monkeypatch + importlib.reload` cycle. Reload-and-assert was avoided because (a) the assertion path is a constant set + regex (deterministic — re-evaluating in the test is logically identical to re-importing), (b) reload would touch the module-level register-probe side effect and risk double-registration. The test verifies the same regex would reject a `"BadID"` injection.
- **`tests/unit/test_schema_validation.py` updated** — not in the story's Files-to-touch list but the 3 Phase-0 schema tests used dict equality with only `{counts, primary}`. The new required fields (`framework_hints`, `monorepo`) made those tests fail. Updated additively (no semantic change to the test intent — same valid/invalid payloads, plus the two new required keys). Same kind of update as the Phase 0 `test_counts_js_fixture` strict-equality adjustment which the story explicitly anticipated under AC-8.

### Lessons (carried to `_lessons.md`)
- L-19: `jsonschema` errors surface as **dotted** `json_path` (`$.probes.x.y`), not JSON-Pointer (`/probes/x/y`). Tests that assert on the path string must match the dotted form.
- L-20: When a slice's required fields grow, every test in the codebase using strict dict equality on that slice (across all phases) breaks at once. Search globally before the GREEN step; better to update them inline than discover them via full-suite failure.
