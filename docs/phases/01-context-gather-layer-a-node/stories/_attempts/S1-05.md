### Deviations from the story (all preserved AC intent; payload-only fixes)

1. **AC-12 mutation expectations widened to `(TypeError, AttributeError)`.**
   The story said "each raises `TypeError`". Empirically (Python 3.13):
   `MappingProxyType.__setitem__` and `.__delitem__` raise `TypeError`,
   but `.update`, `.pop`, `.clear`, `.setdefault` raise `AttributeError`
   ("'mappingproxy' object has no attribute …"). Both shapes mean
   "mutation rejected" — the AC intent. Lesson L-7 applied: the AC's
   behavioral contract ("mutation API blocked") is preserved; the
   payload's exception type was widened. Rationale documented inline in
   the test docstring. Mutation #5 / Mutation #6 in the validator's
   adversary list (a `dict` subclass that masks `__setitem__` but
   permits `.update()`) is still caught — `MappingProxyType` doesn't
   expose `.update()` at all.

2. **`_coerce_sequences` uses `target == tuple[str, ...]`, not `is`.**
   Verified at runtime that `typing.get_type_hints(N)['x'] is
   tuple[str, ...]` returns `False` (each parameterized generic is a
   fresh object); `==` returns `True`. AC-7 is the contract; the
   identity-check would have returned a `list` and tripped the
   parametrized tuple-coercion tests.

### Refactor decisions (Stage 2 design-patterns lens walked)

- **Plugin / Registry-via-discriminator** — `_load_catalog(...,
  schema_subkey: Literal[...])` selects a `$def` from `_LOAD_SCHEMA`.
  Adding a 3rd catalog widens the `Literal` arms only; the kernel body
  is unchanged. AC-20 (`test_kernel_is_closed_for_modification`) and
  AC-21 (`test_schema_subkey_is_literal_typed`) both pass — the same
  plugin-shape framing as `parsers/_io.py` (S1-02/03/04).
- **Smart constructor** — module constants are bound by `_load_catalog`
  only; raw mappings cannot bypass schema validation.
- **Marker exception** — `CatalogLoadError(f"{path}: {detail}")` —
  preserves Phase-0 `test_subclasses_are_markers_only` invariant; AC-13
  `args[0]`-only contract verified.
- **Functional core, imperative shell** — `_coerce_sequences` is pure;
  `_load_catalog` is pure modulo one structlog `info` call (AC-19).
  Module-scope side effect is the import-time binding of the four
  constants — explicit per AC-11 / AC-14.
- **No primitive obsession** — entries are `NamedTuple`s, not
  dictionaries; sequence fields are `tuple[str, ...]` not `list[str]`
  (AC-7 enforces). `parser` is a five-arm `Literal`, not bare `str`
  (AC-8).

- **`# type: ignore[call-overload]` on `entry_cls(**coerced)`** — mypy
  reads the `type[_T]` annotation as the `typing.NamedTuple` factory
  rather than a concrete `NamedTuple` subclass and rejects `**kwargs`.
  Investigated alternatives (`Type[NamedTuple]`, `Protocol[..., _T]`,
  `cast(Callable[..., _T], entry_cls)`); each either fails to remove
  the ignore or pulls extra typing machinery for one call site (Rule 2).
  Single-line targeted ignore with a comment block explaining the
  shape; survives mypy upgrades that fix the underlying issue (the
  ignore would then become an "unused-ignore" warning we'd cleanly
  remove).

  `(TypeError, AttributeError)`** — one-line edit so re-reads of the
  story don't re-litigate this. Not folded here because the story is
