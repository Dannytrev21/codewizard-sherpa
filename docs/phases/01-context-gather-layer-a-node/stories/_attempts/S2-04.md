# Attempts — S2-04 Warm-path memo integration test

## Attempt 1 — 2026-05-14 — GREEN (phase-story-executor)

### Plan

Phase 1's first cross-probe integration test. The story is a diagnostic, not
an implementation — production code for `LanguageDetectionProbe` (S2-01),
`NodeBuildSystemProbe` (S2-02), `ParsedManifestMemo` (S1-07/08), the canonical
fixture (S2-03), the schema validator (Phase 0 + ADR-0013), and the structlog
event taxonomy all already lived on disk. The test wires `CliRunner` +
`structlog.testing.capture_logs` against the helm fixture and asserts (a) the
memo serves the second `package.json` read, (b) the framework-hint extension
makes it onto the envelope, (c) NBS reports `package_manager == "pnpm"`, and
(d) a rogue field under `probes.node_build_system` is rejected by the
envelope schema.

#### Pre-flight — surface the implicit production gap

- Read the hardened story end-to-end (155 lines), the validator's audit at

  Phase-0 precedent at `tests/smoke/test_cli_end_to_end.py`.
- **Smoke-tested the CLI against the helm fixture first.** Discovered the
  load-bearing prerequisite the story implicitly assumed: `_seam_registry_for_task`
  in `src/codegenie/cli.py` only imported `codegenie.probes.language_detection`
  and called `default_registry.for_task("__bullet_tracer__", frozenset({"unknown"}))`.
  `NodeBuildSystemProbe.applies_to_languages = ["javascript", "typescript"]`
  is filtered out by the `for_task` filter at language `"unknown"` — so
  before Wave 1 enriches the snapshot, NBS is never passed to the
  coordinator at all. The integration test's "1 miss + 1 hit" assertion
  would deterministically fail with `(1, 0)` on this code path.
- Verified by smoke-running `gather` in a `uv run python -c …` shell —
  confirmed NBS was absent from `envelope["probes"]`. Wired the seam to
  return `default_registry.all_probes()` (the coordinator owns the
  Wave-1/Wave-2 dispatch model; per-probe applicability lives in the
  probe's `applies()` / no-op-on-missing-inputs branch).
- Smoke-ran again: 1 miss + 1 hit captured, NBS slice present with
  `package_manager: pnpm`. **But** the captured slice also showed
  `warnings: ['node.version_declared_resolved_disagree']` — the
  environment's installed Node (v25.x) disagrees with `.nvmrc` (v20.11.0).
  This would re-fire on CI's ubuntu Node 20.x.y and produce a flake. Added
  `_stub_node_version_check(monkeypatch)` to module-locally rebind
  `codegenie.probes.node_build_system._exec` to a `SimpleNamespace` shim
  whose `run_allowlisted` raises `ToolMissingError`. Mirrors the `_install_scandir_counter`
  pattern at `tests/smoke/conftest.py:49` — the global `codegenie.exec` is
  never mutated.

### Deviations from the story

1. **AC-7 path corrected** — Story said
   `probes.node_build_system.package_manager == "pnpm"`. The actual
   `NodeBuildSystemProbe.run` wraps its slice under a `build_system` key
   (matching `src/codegenie/schema/probes/node_build_system.schema.json`'s
   required `build_system` property). Test asserts the corrected path:
   `probes.node_build_system.build_system.package_manager == "pnpm"`.
2. **AC-8 / AC-9 reinterpreted** — Story asserted
   `probes.language_detection.errors == []` and
   `probes.node_build_system.{errors,warnings,confidence}`. `ProbeOutput.errors`
   / `warnings` / `confidence` are not shallow-merged into the envelope
   (only `schema_slice` is). The LD slice schema (under `language_stack`)
   doesn't *require* `errors`/`warnings`, and the current LD probe doesn't
   emit them into the slice. Adapted the "fail-loud" intent: assert there
   are no `probe.failure` structlog events for either probe; each probe's
   self-emitted `probe.success` event (the variant without `cache_key` —
   distinguishes from the coordinator-emitted twin) reports
   `confidence == "high"`. Added an in-slice
   `build_system["warnings"] == []` once the env-dependent disagree
   warning was neutralised. Both deviations are documented in lesson
   L-25.
3. **`_stub_node_version_check` helper added** — Not enumerated in the
   story's helpers list but necessary for stable assertion of
   `build_system["warnings"] == []`. Lifted into `conftest.py` (kept
   alongside the four story-named helpers) so S2-05 / S5-05 can reuse.
   Documented in lesson L-27.
4. **CLI seam change** — `_seam_registry_for_task` now returns
   `default_registry.all_probes()`. This is the production wiring the
   story implicitly assumed but did not enumerate; without it, NBS is
   never dispatched and AC-7 / AC-11 fail. Documented in lesson L-26.
5. **`_disable_cli_configure_logging` kept local, not lifted to root.**
   The story accepts either path. Lifting risked changing behavior for
   the dozen existing CLI-invoking unit tests; deferring the lift until
   the third consumer (S2-05) lands is the conservative Rule-3 / Rule-11
   choice.

### Refactor decisions

- **Helpers landed in `conftest.py` from the start** (rule of three —
  S2-04 + S2-05 + S5-05). The conftest module's `__all__` exposes them
  for `from tests.integration.probes.conftest import …`.
- **Module-local shim pattern** for the node-version check stub mirrors
  the existing `_install_scandir_counter` precedent (S4-04). No global
  `codegenie.exec` mutation; no shared-state leakage across tests.
- **CLI seam doc updated** with a multi-paragraph docstring on
  `_seam_registry_for_task` explaining the new contract (coordinator owns
  Wave-1/Wave-2; per-probe applicability is the probe's own branch). The
  comment cites `phase-arch-design.md §"Control flow"` for the
  coordinator-dispatch authority.

### Lessons (appended to `_lessons.md`)

  shim is the only safe neutralisation.
