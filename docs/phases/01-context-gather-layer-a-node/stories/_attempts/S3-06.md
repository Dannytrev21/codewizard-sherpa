### Blockers (AC-8 / AC-9 / AC-10 — raw-artifact-budget)

Three independent blockers, each requiring a separate follow-up
outside S3-06's surgical scope:

**B-1 — `NodeManifestProbe` does not emit raw artifacts.** The probe
returns `raw_artifacts=[]` unconditionally (line 478 of
`node_manifest.py`). S1-09's truncation policy in
`cli.py:462–479` is a no-op on an empty list. Adding
`raw_artifacts=[repo.root / selected]` was attempted in this pass;
it broke the AC-7 catalog-invalidation test because the cli writer
names raw artifacts by basename (`raw_path.name`), so the cold-run
write of `.codegenie/context/raw/pnpm-lock.yaml` then matches every
subsequent probe's `rglob("pnpm-lock.yaml")` (the resolver in
`codegenie.cache.keys.declared_inputs_for` uses `rglob`, which is
recursive). `node_build_system`'s warm-run cache key changes
spuriously. The raw_artifact emission was reverted; the underlying
gap remains.

Fix shape (outside S3-06's scope): either (a) namespace the raw-
artifact filename in `cli.py` (e.g., `<probe>.<basename>`), or (b)
add `.codegenie/`-exclusion to `declared_inputs_for` in
`src/codegenie/cache/keys.py`. Either is in `cli` / `cache.keys`,
not in S3-06's test surface.

**B-2 — `os.fstat` monkey-patch is not applicable.** The story
AC-8 prescribes `monkeypatch.setattr(os, "fstat", ...)` to simulate
30 MB without writing it. But `cli.py:466` reads payload via
`Path.read_bytes()` and `raw_truncation.py:88` judges size from
`len(payload)`. Neither calls `os.fstat` on the raw artifact. The
monkey-patch pattern from S3-05 T-9 / S3-01..03 applies to *parser-cap*
tests (where `input_snapshot._fingerprint_from_fd` does call
`os.fstat`), not the *raw-artifact-budget* path. Fixing: either
write a real 26 MB+ file in tmp_path (acceptable per CI disk budget
on tmpfs; not checked in per AC-12), or amend the truncation policy
to size-check via `os.fstat` before reading.

**B-3 — Story-prescribed filename `node_manifest.json` does not
match the writer's basename-derived naming.** Once B-1 unblocks, the
artifact lands at `.codegenie/context/raw/pnpm-lock.yaml`, not
`node_manifest.json`. Story AC-9 needs either an explicit writer-
rename pass or a story-text correction.

### Deviations from the hardened story (preserved here per L-4 / L-7
discipline)

**D-1 — `mocker.spy` → inlined `_CallCounter` helper.** pytest-mock
is not in the dev extras and adding it is out of surgical scope. The
AC-6 intent (call-count + call-through verification of distinct
code paths) is preserved verbatim; the spelling differs. (Similar
to L-3 / L-5 — the typed-exception / behavioral-contract surfaces are
the contract; spellings adapt to shipped infrastructure.)

**D-2 — Spy targets `_yarn._pyarn_parse` / `_yarn._parse_handrolled`,
not upstream `pyarn.parse`.** The `pyarn` package is imported lazily
inside `_pyarn_parse`. Spying on the upstream `pyarn.parse` would
miss the dispatcher's call path. The dispatch endpoints are the
correct observable boundary.

**D-3 — Catalog-loader redirection: seed the catalog INTO the
fixture at the declared-input path, not monkey-patch `_load_catalog`.**
Reason: the cache key participates via `content_hash_of_inputs(
declared_inputs_for(probe, snapshot))`, not via the import-time
`NATIVE_MODULES` constants. Monkey-patching the loader changes the
**probe's behavior** at run-time but does NOT change the
**cache-key fingerprint** the test wants to exercise. The
seed-into-fixture approach exercises the actual cache-invalidation
mechanism the ADR-0006 invariant promises.

**D-4 — Story RED-sketch envelope paths corrected to the
shipped shallow-merge shape.** Story RED sketch references
`nm["primary"]`, `nm["catalog_version"]`, `nm["confidence"]`,
`ctx["probes"]["language_detection"]["primary"]`,
`ctx["probes"]["node_build_system"]["package_manager"]`. The
shipped `cli._seam_shallow_merge` (cli.py:284) does
`probes_block[name] = dict(output.schema_slice)`, so the runtime
envelope shape is:

- `envelope["probes"]["node_manifest"]["manifests"]["primary"]`
- `envelope["probes"]["node_manifest"]["manifests"]["catalog_version"]`
- `envelope["probes"]["node_manifest"]["manifests"]["warnings"]`
- (no `confidence` — that's a `ProbeOutput` field, not in
  `schema_slice`, so the shallow-merge loses it)
- `envelope["probes"]["language_detection"]["language_stack"]["primary"]`
- `envelope["probes"]["node_build_system"]["build_system"]["package_manager"]`

The `confidence: high` AC-3 assertion is recovered from
`manifests.warnings == []` (clean fixture) / `lockfile.multi_present
in manifests.warnings` (stray-lockfile arm). Same behavioral
contract, observable from the surviving envelope shape. (Similar to
L-21 from prior stories — `schema_slice` is the merge unit.)

**D-5 — `language_detection.primary` is `None` for the
manifest-only fixture, not `"javascript"`.** The fixture ships zero
source files; the probe walks file counts only, so `primary=None`
is the correct deterministic output. AC-4's "multi-probe
end-to-end" intent is pinned by `node_build_system.package_manager
== "pnpm"` + the language_detection slice's structural presence
(not its `primary` value). Adding a sentinel `.js` file to bump
`primary == "javascript"` would have widened the fixture's purpose;
the slice-presence assertion is the minimum-invasive pin.

**D-6 — AC-3 tautology kill is recovered from
`primary.direct_dependencies.runtime == 3`, not from the resolved-
packages list.** The probe does not surface a separate "all resolved
packages" list in its schema slice (privacy-by-default; only catalog
hits + summary counts). `direct_dependencies.runtime == 3` confirms
all three of `bcrypt`/`sharp`/`lodash` are read from `package.json`;
a probe that hardcoded `{"bcrypt", "sharp"}` would report 2, not 3.

### Lessons (carry forward to next stories)

- **L-35 — `cli`-side raw-artifact filename is the basename of the
  emitted path; basenames collide across probes.** Adding raw-
  artifact emission to a probe that shares basenames with another
  probe's `declared_inputs` (or with another fixture file) causes
  spurious cache invalidations on warm runs. The fix shape lives in
  `cli.py` (namespace the filename) or in `cache.keys.declared_inputs_for`
  (exclude the `.codegenie/` output namespace from rglob); not in
  the probe itself.

- **L-36 — `declared_inputs_for` uses `rglob`, not `glob`.** When
  reasoning about probe cache-key contributors, remember the glob is
  recursive. A `"pnpm-lock.yaml"` literal in `declared_inputs`
  matches every `pnpm-lock.yaml` anywhere in the repo tree, including
  cli-written artifacts under `.codegenie/`. This bites once and is
  easy to miss because the glob *looks* like a literal filename.

- **L-37 — Catalog-edit cache invalidation requires the catalog
  file IN THE FIXTURE at the declared-input relative path.** The
  `declared_inputs` entry `"src/codegenie/catalogs/native_modules.yaml"`
  is matched against the repo root via `rglob`, not against the
  codegenie install root. So a fixture without that path has *no
  catalog contribution to the cache key* — the ADR-0006 invariant
  cannot be exercised by monkey-patching the in-process loader.
  Seed the catalog into the fixture at the declared-input path; edit
  it there.

- **L-38 — pytest-mock is not in dev extras; inline a `_CallCounter`
  for spy-style tests.** A 6-line `__call__` + `monkeypatch.setattr`
  helper is the right shape for stories that prescribe `mocker.spy`.
  Don't add `pytest-mock` to the dev extras just to satisfy a story
  spelling; the spy *behavior* (call-count + call-through) is the
  contract, not the spelling.

### Deviations

**D-7 — AC-9 filename.** Story AC-9 prescribed
`.codegenie/context/raw/node_manifest.json`; the writer uses
`raw_path.name` so the actual on-disk path is
`.codegenie/context/raw/pnpm-lock.yaml`. The wrapper's *content* is
JSON (truncation marker) and AC-9's load-bearing invariants
(`json.loads` succeeds, `__truncated_at_budget__ is True`, byte counts)
pin against the content, not the extension. Renaming on the writer
side would widen S3-06 scope (Rule 3 — surgical changes) and break
the basename-as-leaf chokepoint that ADR-0008 §Sanitizer relies on
(the writer's `_assert_safe_name` is the chokepoint for raw artifact
leafs — namespacing on top of it adds a second naming source without
a load-bearing reason).

### Lessons (carry forward)

- **L-39 — `.codegenie/` is the output namespace; exclude it from any
  input fingerprint resolver.** `declared_inputs_for`'s `rglob` would
  otherwise capture cli-written outputs and cross-contaminate cache
  keys across probes. The fix is a one-line `rel.parts[0] != ".codegenie"`
  check, not per-probe basename namespacing. Future readers of
  `cache/keys.py`: if you add another resolver, add the same filter.

- **L-40 — `os.fstat` + bounded `os.read` is the right shape for
  size-bounded artifact reads.** Reading via `Path.read_bytes()` loads
  the full file into memory before truncation — defeats the budget at
  portfolio scale. The CLI's raw-artifact loop now reads at most
  `budget_bytes` regardless of file size and reports the true original
  size to the truncation helper via `original_bytes=...`. Short-read
  semantics are handled with an accumulation loop (POSIX `os.read` on
  regular files normally returns all requested bytes, but the loop is
  defensive against future tmpfs / NFS surprises).

- **L-41 — Story-prescribed paths can drift from writer-side naming.**
  When a story AC reads a specific filename, verify that filename
  matches the writer's actual chokepoint. The validator's HARDEN pass
  cannot catch every writer-vs-story drift; deviations like D-7 are
  acceptable when the load-bearing invariant is content-shaped, not
  path-shaped.
