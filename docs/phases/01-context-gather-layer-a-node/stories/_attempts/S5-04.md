### Decisions / divergences

- **turbo.json uses v2 `tasks` shape**, not v1 `pipeline`. The
  content-check predicate `_turbo_json_minimum_shape` accepts either
  (AC-MR-4 explicitly allowed). Picked v2 because that's the current
  shape at land-time per `turbo.build/schema.json`.
- **pnpm-lock.yaml uses an `importers:` stanza** (per pnpm v6
  monorepo lockfile convention) listing the root + both workspace
  paths. This is the smallest valid lockfile that survives YAML
  round-trip and pins the `pnpm` lockfile-precedence path in
  `NodeBuildSystemProbe`. Hand-authored — never ran `pnpm install`
  per the implementer note in §"Notes for implementer".
- **AC-NN-6 ("Node-only slice keys ABSENT in envelope")** is a smoke
  check observable, not a shape-test assertion in S5-04 — the story
  body explicitly says "structural per-slice assertions land in
  S5-05". Current CLI dispatch
  (`cli.py:_seam_registry_for_task`) bypasses the registry's
  `for_task` language filter and runs every probe (each emits a
  minimal slice when its inputs are missing). S5-05's integration
  test will need to either assert presence-with-null contents OR
  request a CLI behavior change. Flagged for the S5-05 implementer
  rather than absorbed here — S5-04's scope is fixtures + shape
  tests.
