### What didn't work the first time

- **Single-line docstrings tripped E501 (>100 chars).** First green pass
  used one-line docstrings copied verbatim from the story. Ruff failed
  with 6 line-too-long errors. Resolved by wrapping each docstring across
  2–4 lines while preserving every required substring (`parsers`,
  `catalogs`, `hard fail at CLI startup`). This is a stylistic delta from
  the story's literal example, not a contract change — the docstring
  *content* tests (slug presence + hard-fail substring) remained green.

### Lesson

When a story includes verbatim code examples with prose-length docstrings,
check the project's line-length lint config before pasting. Phase 0's
`ProbeBudgetExceeded` already wraps its docstring across multiple lines —
match that pattern, not the example formatting in the story file.
