### Deviations from the story

1. **Test stubs are `async def`, not sync lambdas.** The story's preamble
   uses `lambda argv, **kw: types.SimpleNamespace(...)` to stub
   `codegenie.exec.run_allowlisted`. But that function is `async def` in
   this codebase (`src/codegenie/exec.py:175`) — awaiting a sync-returned
   `SimpleNamespace` would raise `TypeError`. The stubs here are
   `async def` returning the same `ProcessResult`-shaped object; the seam
   contract (AC-12 / T-15: monkeypatch `codegenie.exec.run_allowlisted` and
   verify the probe routes through it) is fully preserved. This is a
   test-shape adjustment, not an AC weakening.
2. **AC-6c exception set widened to match the real wrapper.** The story
   lists `(FileNotFoundError, TimeoutExpired, ExecError)` but the real
   `exec.run_allowlisted` raises `ToolMissingError` (binary not on PATH),
   `ProbeTimeoutError` (timeout), and `DisallowedSubprocessError`
   (allowlist miss; never expected for `node` but defensive). The probe
   catches the **actual** wrapper exceptions plus a non-zero `returncode`
   sentinel for exec-error, and tests cover each branch separately
   (T-6a..T-6d). `subprocess.TimeoutExpired` from the story is not raised
   by the real wrapper, so it would have been an unreachable catch arm.
3. **An additional `probe.tsconfig.parse` log line was added** so the
   `parser_kind` field observability anchor (T-20) emits even when no cap
   fires. The jsonc parser emits `parser_kind` only on cap events; the
   story's prose ("Every parse-related log line includes the `parser_kind`
   field") is honored by a per-walk log line that always carries
   `parser_kind="jsonc"`.
4. **Sub-schema `errors` field is optional, not required.** Story AC-10
   names both `warnings[]` and `errors[]` as fields the sub-schema
   constrains; it does not say `errors` is *required*. The schema here
   only requires `warnings` (always emitted) and leaves `errors` as an
   optional forward-compatible field on the slice. Typed-exception IDs
   land on `ProbeOutput.errors` per AC-14 (the envelope, not the slice) —
   this is consistent with how Phase 0's `language_detection` slice
   handled the same trade-off.

### Lessons
- L-21: `codegenie.exec.run_allowlisted` is `async def`. Any story that
  prescribes a sync `lambda` stub for it needs adjustment to `async def
  _stub(...)`. The seam contract (monkeypatch the attribute on
  `codegenie.exec`) is correct; only the stub shape needs to be
  awaitable. Apply to: any future probe in S3-05/S4-01/S4-02/etc. that
  invokes an allowlisted binary.
- L-22: The wrapper's exception surface is **not** `(FileNotFoundError,
  TimeoutExpired, ExecError)` — those are the story's literal nominal set.
  The real set is `(FileNotFoundError, ToolMissingError, ProbeTimeoutError,
  DisallowedSubprocessError)` plus non-zero `returncode`. Catch the real
  set; the story's wording is guidance.
- L-23: `parser_kind` observability needs an emit *at parse time*, not
  only on cap events. The jsonc parser emits `parser_kind` exclusively on
  `probe.parser.cap_exceeded` — happy-path callers see no `parser_kind`
  unless they log it themselves. When a story's AC depends on the field
  being present on a normal parse path, the probe (not the parser) should
  emit the anchor line.
