### Deviations from the hardened story

1. **AC-6 — `_stat_snapshot` excludes `<root>/.codegenie/`.** The
   story prescribes a fixture-immutability invariant via
   `_stat_snapshot(repo)` pre- and post-gather. Without an exclusion
   filter the assertion always fires because the gather *legitimately*
   writes `repo-context.yaml`, `runs/<id>.json`, `cache/blobs/...`,
   and `cache/index.jsonl` under `<root>/.codegenie/`. The invariant's
   intent is to catch *input* drift, not output writes. The helper
   docstring now states "regular file under `root`, **excluding** the
   codegenie output namespace `<root>/.codegenie/`". Documented in
   lesson L-28.
2. **AC-14 / AC-15 — slice-content fail-loud signals split.** The
   story asserts `probes.language_detection.errors == []`,
   `probes.language_detection.warnings == []`,
   `probes.node_build_system.errors == []`,
   `probes.node_build_system.warnings == []`, and
   `probes.node_build_system.confidence == "high"` *at the top of each
   probe's envelope entry*. The envelope only carries each probe's
   `schema_slice` shallow-merged under `envelope.probes.<name>`
   (S2-04's lesson L-21 / L-25 — `ProbeOutput.errors/warnings/confidence`
   are not merged). The fail-loud intent is preserved by replacing
   top-level checks with: (a) in-slice
   `build_system["warnings"] == []` (the NBS sub-schema includes
   `warnings`); (b) zero `probe.failure` events in `warm_logs` for
   either probe; (c) zero probe-internal `probe.success` events on
   warm (variant pin — the probe never ran). This matches the same
   deviation S2-04 documented.
