### Stage 2 — Implementation

#### Architectural fix surfaced by AC-NN-3 + AC-NN-5

The story's AC-NN-3 ("Node-only probes ABSENT on a Go repo") and
AC-NN-5 ("audit record records `Skipped`") presupposed a
language-applicability filter that did **not** exist on disk.
Running gather against the `non_node_go` fixture before any changes
showed all six probes ran-and-produced-empty slices — `node_build_system`,
`node_manifest`, and `test_inventory` were present with no-evidence
content, not absent.

Root cause: every Phase-1 probe declared `tier = "base"` (the
coordinator's prelude wave runs them all concurrently against the
ORIGINAL snapshot with `detected_languages = {}`), so the per-probe
`applies_to_languages` filter never gated anything. The architecture
docstring's intent ("base probes run first; remaining probes dispatch
against the enriched snapshot") was implemented for the partition but
not for the filter.

Resolution (minimum-scope):

1. `NodeBuildSystemProbe` / `NodeManifestProbe` / `TestInventoryProbe`:
   `tier = "base"` → `tier = "task_specific"`. Each gets an
   `applies()` override that consults the new
   `codegenie.probes.language_filter._admits_node_project` predicate.
2. New module `src/codegenie/probes/language_filter.py` — two pure
   helpers (`_admits_languages`, `_admits_node_project`). The
   `package.json`-at-root fallback admits greenfield Node repos whose
   `language_detection` counts no source files yet (e.g.,
   `node_pnpm_native` fixture) — without that fallback, S3-06's
   cache-invalidation test would fail because the catalog-only
   fixture has zero `.js`/`.ts` files.
3. Three corresponding unit-test pins (`test_node_build_system.py`,
   `test_node_manifest.py`, `test_test_inventory.py`) updated to
   `task_specific`.

#### CLI shallow-merge fix (surfaced by AC-ERR-1)

The first run of `test_failure_isolation.py` failed with
`exit_code=3 schema_invalid`: a failure-isolated probe carries
`schema_slice = {}` (per `coordinator._build_failure_output`), which
`_seam_shallow_merge` previously wrote as `envelope.probes.deployment
= {}` — violating the deployment sub-schema's `required: ["deployment"]`
contract. Per ADR-0010 ("Layer A slices optional at envelope"), the
empty slice should be **omitted**. Fixed `_seam_shallow_merge` to
`continue` when `output.schema_slice` is empty. The probe's failure
remains visible via the audit run-record's `exit_status == "error"`
row.

### Stage 4 — Reflection

**What worked.** The conftest's existing Open/Closed seam
(`WARM_PATH_CACHE_HIT_PROBES`) flowed cleanly to the all-Phase-1
extension once aliased to `PHASE_1_PROBE_NAMES` — zero edits to the
existing two-probe test function bodies, exactly as the seam was
designed for. The three pure shape-assertion helpers compose
naturally.

**What needed correction mid-flight.**

- First-draft AC schemas conflated `ci.providers` (plural) and
  `deployment.manifests` with actual schema field names — pinned to
  reality (`ci.provider` singular; `deployment.type` etc.).
- AC-PR-2's `Ran | CacheHit | Skipped` union confused with an enum —
  asserted the three load-bearing audit-record fields together
  instead.
- The story expected the metamorphic cache-miss test to cover all six
  probes; CI and Deployment declare inputs orthogonal to
  `package.json`, so they stay cache-hit. Surfaced via the
  `_PACKAGE_JSON_DEPENDENT_PROBES` subset.
