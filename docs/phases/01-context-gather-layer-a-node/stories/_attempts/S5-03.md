### Refactor decisions (design-pattern lens)

- **Newtype / domain primitive:** `GatherResult` already uses a
  frozen dataclass — added two `str` / `dict[str, str]` fields without
  introducing new types. The test-helper API is a single seam; no
  primitive obsession introduced.
- **Strategy / dispatch:** the three test files mirror the three
  defenses (parser, probe, CLI; kustomize-overlay; jsonc-stripper).
  Per the story's three-layer-observation pattern, each defense gets
  one test file as the observation surface. No premature
  helper-extraction (Rule 2, three-similar-lines threshold not met —
  each sentinel pattern has one consumer).
- **Anti-patterns avoided:** no `pytest.mark.asyncio`, no shared
  conftest fixture (single-call sites; would be premature
  abstraction), no LLM, no untyped `Any` smuggling at module
  boundaries.
- **Phase-2 design observation (Notes-for-implementer §Phase-2):**
  `_walk_extends` returns `tuple[dict, list[str]]` — anaemic; a
  tagged-union `_TsconfigOutcome = Resolved | Cycle | DepthExceeded |
  Absent` would make illegal states unrepresentable. Logged here as a
  follow-up against S2-02; out of scope for S5-03.
