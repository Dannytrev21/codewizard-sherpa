### Lessons
- **L-24:** `codegenie gather <fixture-dir>` writes `.codegenie/` into
  the analyzed directory by default. Any future fixture-shape story
  that runs `gather` for an AC-22-style smoke check must either
  (a) clean up the `.codegenie/` afterward, or (b) run with an
  `--output` redirect outside the fixture. Otherwise the same closed-set
  invariant (AC-14) that this story enforces will fire on the very next
  test run. Apply to: S5-04 (`node_monorepo_turbo/` + `non_node_go/`),
  S2-05 (cache-hit-on-real-repo), S5-05 (e2e).
- **L-25 (precedent reinforced from S2-01/S2-02):** The
  `_FILE_SPECS: tuple[_FileSpec, ...]` Open/Closed-at-file-boundary seam
  works *exactly* the same for fixture-shape tests as for probe-precedence
  tables — the parametrized test bodies stay frozen across additions.
  Re-evaluate kernel extraction at S5-04 (third + fourth consumers per
  rule-of-three). The premature-abstraction guard noted in the story's
  Notes-for-implementer held: no kernel needed yet.
