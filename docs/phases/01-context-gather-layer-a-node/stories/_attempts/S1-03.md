# Attempt log: S1-03 — `safe_yaml` parser with `CSafeLoader` + `load_all` + alias-safe depth walker

## Attempt 1 — 2026-05-14 — SUCCESS

**Approach:** Took the lift-or-create path the hardened story prescribes —
created `parsers/_io.py` (O_NOFOLLOW + size-cap primitive) and
`parsers/_depth.py` (id()-memoized walker), refactored `safe_json.py`
to consume both, then built `safe_yaml.py` as a thin strategy on top.
Wrote 46 tests (the 20 named in the story plus parametrize expansions
and three tightening additions) before the first line of production
code; ran them red, implemented green in one pass.

**ReAct cycles:** 1 (no retries needed — Stage 3 green on first pass).

**What worked:**
- **Story-prescribed lift was the right call.** Putting `_io.open_capped`
  and `_depth.assert_max_depth` in shared modules made `safe_yaml.py`
  ~80 lines (mostly docstring) and meant the alias-amplification
  defense lives in exactly one place. Future `safe_toml`/`safe_xml`
  adds a new file + new `parser_kind` literal; zero edits here.
- **`id()` memoization in the walker = O(physical nodes), not O(logical
  visits).** AC-12's ten-anchor bomb (logical 10^10 visits) completed
  in <100 ms with <50 MB delta. A naive recursive walker hangs the
  test runner — running the test in CI is the test (per the story's
  "killer mutation" framing).
- **`structlog.testing.capture_logs` works without configuring the
  default processor chain.** The harness already had `structlog`
  pre-configured globally; capture_logs returns a list of event dicts
  with the structured fields intact (`cap_kind`, `cap`, `path`,
  `parser`, `parser_kind`).
- **Module-level CSafeLoader guard via `getattr(yaml, "CSafeLoader",
  None) is None` → `ImportError`.** `monkeypatch.delattr(yaml,
  "CSafeLoader") + importlib.reload(safe_yaml)` is the only way to
  exercise the guard since CSafeLoader is normally present;
  `monkeypatch.undo() + importlib.reload(safe_yaml)` restores state
  for downstream tests.
- **Generator shape via `def _generate(...): yield`.** Returning the
  generator from `load_all` makes `inspect.isgenerator` true; the per-
  doc walker invocation is inside the generator body so a valid first
  doc surfaces before a later doc raises (AC-16).

**What didn't:**
- One regression in `tests/unit/parsers/test_safe_json.py::
  test_short_read_translates_to_malformed_json` after the lift — the
  old safe_json had an explicit `len(data) != size: raise
  MalformedJSONError("…short read…")` check; the lifted
  `_io.open_capped` returns whatever bytes were read and lets the
  parser's decoder reject. Behavioural contract preserved (short read
  still surfaces as `MalformedJSONError`), but the test's literal
  "short read" message check broke. Updated the assertion to drop the
  message-substring check and added a comment naming the S1-03 lift as
  the cause. Did NOT fold "short read" handling back into the
  primitive — the parser-level decoder failure is the cleaner path
  (Rule 2: no special-case handling that the natural parse error
  already covers).

**Root cause of one-test regression:** Refactoring an inline I/O path
into a shared primitive is a contract change at the message-text level
even when the typed-exception contract is preserved. Tests that pin
exception message substrings are brittle to such refactors; tests that
pin exception **types** are not.

**Lesson for next attempt:** When lifting a defense into a shared
primitive, scan for downstream tests asserting message *text* and
relax to type-only assertions where the test's underlying intent is
"the right typed exception fires" rather than "this exact message
fires."
