### What didn't / surprised me

- **`ruff format --check` re-flagged one long-line in the test file**
  after the new tests landed. Auto-formatting was an immediate fix (one
  long string literal split across two lines on the `ALLOWED_BASE_PY_IMPORTS`
  one-liner). No semantic change.
- **`__annotations__` repr difference vs. test substring matching.** The
  annotation for `parsed_manifest` renders as
  `collections.abc.Callable[[pathlib._local.Path], collections.abc.Mapping[str, typing.Any] | None] | None`
  — substring `"Callable"` and `"Mapping"` both appear, so the test passes.
  If a future Python version renames `collections.abc`, the annotation
  test would still pass because the public-name substring is invariant.

### Lessons (cross-story candidates)

- **Hardened stories pay off in single-attempt success.** S1-06's validator
  pre-decided every contested call (`InputFingerprint` location, the
  `Mapping[str, Any]` vs `JSONValue` decision, the script-vs-test home for
  the sentinel). The implementer's job collapsed to mechanical execution.
  When validator output is this thorough, the executor should treat the
  story as the contract, not a starting point. (See `_lessons.md` L-11.)
- **NamedTuple > frozen dataclass for value-keyed sets.** When the only
  reason for a "frozen dataclass" is membership in a `frozenset`, prefer
  `NamedTuple` — auto-hash, auto-equality, smaller memory footprint,
  pickling-friendly. Frozen-dataclass with `eq=True` is the same thing
  with more boilerplate. (`_lessons.md` L-12.)

### Refactor decisions (design-pattern lens)

Walked the refactor lens at GREEN time:

- **Newtype / smart constructor for `InputFingerprint.path`.** Story Notes
  flag `path: NormalizedAbsPath` as a Phase-2 sharpening opportunity
  (rule-of-three deferred). Not introduced. Recorded in story Notes.
- **`ParsedManifestCapability` Protocol** instead of bare `Callable`.
  Deferred until S1-07 lands the concrete memo so the Protocol's pull
  weight is observable. Bare `Callable` is right-sized for a single seam.
- **Tagged union / sum-type opportunity.** `parsed_manifest` and
  `input_snapshot` are both `Optional[T]` with a "fall back to direct
  read" branch in every consumer. Not unified into a sum type because
  (a) the two fields serve different purposes (one is a callable, one is
  data), (b) collapsing them would couple unrelated consumer codepaths.
  Recorded.
- **No anti-patterns introduced.** The amendment is additive, with `None`
  defaults; no hidden state, no primitive obsession beyond the already-noted
  `path: str` decision, no pure-impure tangle (`base.py` stays stdlib-only).
