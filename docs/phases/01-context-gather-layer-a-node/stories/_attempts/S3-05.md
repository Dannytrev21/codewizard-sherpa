### What changed in this pass

- **`tests/unit/probes/test_node_manifest.py`** — three failing
  sub-schema tests were wrapping the payload at the wrong level. The
  schema's slice root is `{"manifests": {...}}` (consistent with
  `node_build_system.schema.json`'s `{"build_system": {...}}` and the
  `K-2` validation finding "All assertions use
  `out.schema_slice['manifests']`"). The fixtures injected extras
  into the inner `manifests` dict but passed the un-wrapped dict to
  `jsonschema.validate`, which then failed at the outer
  `additionalProperties: false` instead of the intended nested
  pointer. Wrapped each fixture under `{"manifests": ...}` so the
  pointer assertions pin the intended nested rejection sites.
- **`src/codegenie/probes/node_manifest.py`** — mypy `--strict` flagged
  two unreachable defensive guards in `_flatten_npm` and
  `_flatten_yarn` (the param type union `_NpmLock | Mapping[str, Any]`
  is always a `Mapping`). Inlined the `isinstance(parsed, Mapping)`
  check at the first `.get(...)` site, matching the pattern already in
  `_flatten_pnpm`. Behavior unchanged — both helpers still return `{}`
  when handed a non-mapping at runtime (now via the inlined guard).
- **`tests/integration/probes/test_language_detection_warm_path.py`**
  — the S2-04 invariant was "exactly 1 memo hit across the two
  pre-S3-05 probes". With `NodeManifestProbe` joining
  `LanguageDetectionProbe` and `NodeBuildSystemProbe` as a third
  `package.json` consumer, the expected hit count rises from 1 to 2
  (one miss + N-1 hits, where N = consumer count). Generalised the
  assertion against `len(package_json_consumers) - 1` so the test
  scales as future probes join. The miss-then-hit invariant — the
  load-bearing S1-07 memo contract — is preserved.
- **New coverage tests** — added three high-value runtime-path tests
  (`test_malformed_package_json_emits_typed_error_and_low_confidence`,
  `test_missing_package_json_emits_empty_primary_high_confidence`,
  `test_package_json_memo_hit_short_circuits_safe_json_load`) plus one
  defensive-guard test
  (`test_flatteners_return_empty_on_non_mapping_inputs`). These cover
  the `pkg is None` branch, the `not exists` branch, the memo-hit
  short-circuit, and the inlined defensive guards in the three
  flatteners — taking the per-module line coverage from 85 % → 92 %.
- **Polluted fixture cleanup** — removed `.codegenie/` and
  `.gitignore` files that an earlier interactive `codegenie gather`
  run had left under five `tests/fixtures/*` directories.
  `test_fixture_node_typescript_helm_shape.py` enforces no-pollution
  invariants and was failing on them. The fixture-shape tests now
  pass.

### Lessons captured (for `_lessons.md` future appends)

- When resuming a partially-landed story, run `pytest --no-cov` first
  to surface the gap quickly. The default `--cov-fail-under=85` gate
  masks individual test failures behind a coverage report.
- A `manifests`-style wrapper in the slice schema (mirroring
  `build_system`, `language_stack`) is the established convention
  — the story prose's `/probes/X/primary/...` JSON-Pointer form is a
  pre-hardening artifact and should be treated as advisory; the
  validation file's behavioural contracts override.
- Adding a new probe that reads `package.json` shifts the
  warm-path-memo expected counts. Tests that pin
  "exactly 1 hit" should scale with the consumer list.
