### What didn't, the first time

- **Test 5 (timeout escalation) failed first run.** My initial
  `_escalate_and_kill` tried to be "smart" — SIGTERM, `await
  proc.wait()` with a grace timeout, *only then* SIGKILL if the wait
  timed out. The mock's `proc.wait()` AsyncMock returns immediately, so
  the wrapper saw the child as "exited during grace" and never called
  `kill()`. Test asserted `kill.call_count >= 1` → AssertionError.
  Resolution: re-read the story's implementation-outline step (vi) —
  "`proc.terminate()`, wait ~100 ms, `proc.kill()`, `await proc.wait()`,
  raise `ProbeTimeoutError`" — the escalation is **unconditional**.
  Switched to `asyncio.sleep(_SIGTERM_GRACE_S)` then unconditional
  `proc.kill()`. Calling `kill()` on an already-exited process is
  harmless and gives the test a deterministic spy assertion.
  Lesson: when a story's TDD spec and implementation-outline both
  prescribe a literal call sequence, do not "optimize" around the spec
  — the test pins the literal sequence for a reason (Phase 7's
  coordinator-cancel needs the SIGKILL invocation to be observable).
- **Forbidden-patterns pre-commit hook flagged the module docstring.**
  My docstring described the chokepoint by saying it "blocks
  `subprocess.run(..., shell=True)`" — the literal phrase matched
  `scripts/check_forbidden_patterns.py`'s `subprocess.run(..., shell=...)`
  and `shell=True` rules. The hook scans the script's own rule table
  via the `exclude:` rule but not source docstrings. Resolution:
  paraphrased the docstring to describe the property without naming
  the forbidden literal. Lesson: the hook is a regex over bytes, not
  an AST — even *describing* a banned pattern in prose trips it.
  Avoid the literal in source comments/docstrings.

### Lesson for next time

- When a chokepoint test asserts an "unconditional sequence" of side
  effects (signal A, then signal B, regardless of intermediate state),
  the implementation must be unconditional too. "Optimal" early-exit
  defeats observability. The Phase 7 coordinator-cancel story will
  iterate `_RUNNING_PROCS` and call `proc.kill()` itself — the test
  pins the protocol the coordinator will rely on.
- Docstrings in `src/codegenie/` are subject to the forbidden-patterns
  hook. Reference banned constructs by description ("shell-enabled
  subprocess call"), never by literal token (`shell=True`).
