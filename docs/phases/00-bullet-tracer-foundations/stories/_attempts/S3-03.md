### Key decisions and lessons

1. **Longest-prefix-wins via alternation order, not regex lookbehind.** Python's `re` module evaluates `|` left-to-right and stops at first match. Placing `re.escape(str(repo_root))` first in the alternation means that when `repo_root=/Users/danny/repo` matches both the repo prefix and `/Users/<u>/`, the longer (repo) prefix wins automatically. No lookbehind, no separate sort pass — the regex engine does the work. Verified by `test_pass2_repo_under_users_prefers_repo_prefix`.

2. **Per-call regex compile is intentional, not waste.** The path regex embeds `re.escape(str(repo_root))` so it varies per call. The story's "lazy impl mutant" warning about anchored regex (`^(...)`) was a real concern — `re.compile(..., re.MULTILINE)` would have been an attractive over-engineering; sticking with non-anchored `re.sub` keeps the embedded-mid-string scrub correct and the no-leak invariant test (AC-12) catches any drift.

3. **Two story-internal test bugs surfaced during red→green.**
   - **`tokens_per_line` in the benign-key matrix.** The canonical `SECRET_FIELD_PATTERN` from `coordinator.validator` matches any name containing `token` (per ADR-0010 §Tradeoffs: prefer false positives over silent leaks). The story's benign-key matrix as written would have failed every benign-key test for `tokens_per_line`. Removed that entry and added an inline comment citing ADR-0010 §Tradeoffs.
   - **`_iter_strings` walks dict keys, but `test_pass2_walks_arbitrary_depth` asserted no keys appeared.** The helper as written by the story iterates `dict.items()` and walks both `k` and `v`; the assertion `flat == ["src/a.js"]` would never pass because every nested dict's `"l"` key (and the outer `"root"`) appear in the flat list. Replaced the equality assertion with the intent (`"src/a.js" in flat`, no `/tmp/.../src/a.js` substring anywhere) — encodes WHY the test exists (Rule 9) and survives the helper's actual behavior.
   - Surfaced via the `phase-story-validator` follow-up channel: these are story-internal bugs in the TDD plan, not implementation drift.

4. **`yaml.CSafeDumper` fallback indirection via `_import_csafe_dumper()`.** Tests need to simulate libyaml-missing without monkeypatching `yaml` itself (which would break unrelated tests in the same process). Wrapping `yaml.CSafeDumper` in a one-line function gives `monkeypatch.setattr(writer_mod, "_import_csafe_dumper", no_csafe)` a clean seam. `_csafe_warned` is a module-level mutable `bool`; once-per-process semantics fall out without any test pollution because `monkeypatch.setattr(writer_mod, "_csafe_warned", False, raising=False)` resets the flag at the start of the only test that exercises the fallback path.

5. **Symlink refusal happens before raw-name validation and before `mkdir`.** AC-20's "no `.tmp` produced" check confirms this — refusing a symlinked `output_dir` must short-circuit before any file or directory is created. The implementation orders: (1) symlink-refuse `output_dir`, then `output_dir/raw/` and `output_dir/repo-context.yaml` if `output_dir.exists()`; (2) validate every raw-artifact name; (3) `mkdir`; (4) write raws; (5) write envelope; (6) recursive chmod. The `output_dir.exists()` guard is necessary because a non-existent output_dir is a valid happy path on first run.

6. **Recursive chmod walks `output_dir` post-write, not per-file inline.** Per ADR-0011 edge case #6, files restored by `actions/cache` may have lost their tight modes; only re-chmodding the file just written misses sibling state. Calling `_fix_modes_recursively(output_dir)` as the last step in `write()` covers both the newly-written tree and any pre-existing children. Verified by `test_writer_fixes_preexisting_loose_modes` which creates a `0644` file inside `output_dir/raw/` before calling `write` and asserts it is `0600` afterward.

7. **`types-PyYAML` gap between local and pre-commit mypy.** The pre-commit hook lists `types-PyYAML` in `additional_dependencies`, but `[project.optional-dependencies].dev` did not — so `make typecheck` (after `make bootstrap`) would fail on a fresh contributor checkout the moment `src/codegenie/output/writer.py` lands. Added the stub to `dev` extras with a one-line citation comment. Story didn't list `pyproject.toml` as a file to touch, but AC-26's "`mypy --strict` clean" gate transitively requires the stub.
