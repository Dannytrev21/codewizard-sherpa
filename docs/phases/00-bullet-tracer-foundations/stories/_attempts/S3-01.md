### What worked

- **`_ProbeLike` Protocol bridges the frozen ABC.** The `Probe` ABC in
  `src/codegenie/probes/base.py` is byte-for-byte `localv2.md ┬¦4` (ADR-0007)
  and does NOT declare `version` as a class attribute ŌĆö `registry.py`'s
  docstring spells this out as a "convention, not part of the frozen ABC."
  Reading `probe.version` from `cache/keys.py` would have failed
  `mypy --strict` with `"type[Probe]" has no attribute "version"`. Defining
  a local `Protocol` with `name`, `version`, and `declared_inputs` decouples
  the cache from the ABC's frozen state ŌĆö surgical extension by addition
  (CLAUDE.md Rule 11) rather than amending the snapshot. Synthetic test
  probes (plain classes, no inheritance from `Probe`) satisfy the Protocol
  structurally.
- **`_key_meta: dict[str, tuple[str, str]]` stash populated by `key_for`.**
  The story sketches `put(key, output)` with no probe argument, but the
  index record schema requires `probe_name` and `probe_version`. Resolving
  this with a per-store dict populated by `key_for(probe, snapshot, task)`
  keeps the public `put` signature exactly as the story sketches AND
  pins the discipline that the coordinator must compute the key before
  storing the output (the coordinator naturally does this; tests follow
  the same sequence). A `put` without a preceding `key_for` raises
  `CacheError` with a pointer to the missing call ŌĆö fail-loud (CLAUDE.md
  Rule 12) rather than silently writing `""` for the probe identity.
- **Monkeypatching `_SCHEMA_DIR` for the invalidation-scope tests.** Tests
  rebuild a minimal envelope + `probes/<name>.schema.json` tree under
  `tmp_path` and `monkeypatch.setattr(keys_mod, "_SCHEMA_DIR", schema_dir)`
  to point the resolver there. This isolates Gap-1's regression test from
  the real installed package: bumping a synthetic probe's `$id` in
  `tmp_path` doesn't risk drifting any real probe's cache.
- **`structlog.testing.capture_logs()` over `caplog`.** The story sketch
  used pytest's `caplog`, but `codegenie.logging`'s `PrintLoggerFactory`
  writes to stderr and bypasses stdlib logging ŌĆö `caplog` would not capture
  any events. Structlog's `capture_logs` context manager temporarily
  redirects the logger to an in-memory list of event dicts; each test
  asserts on the `event` field directly. Cleaner than parsing stderr JSON.
- **Atomic-write test via `mock.patch("codegenie.cache.store.os.replace")`.**
  Patching the imported `os.replace` (not the global) and asserting that
  no `*.json` file appears under `blobs/` afterwards verifies the
  invariant from `phase-arch-design.md ┬¦Failure behavior`: a partial
  blob is never visible at the final path.

### Surprises / things to flag for future stories

- **Record-size guard input.** The story AC-12 text says "a test crafts
  a probe with an oversized `schema_slice`," but the index record schema
  does NOT include `schema_slice` (that field lives in the BLOB, not the
  index). The semantically correct interpretation: any input that bloats
  the serialized record over 4096 B should trigger the guard. The test
  uses a `version="v" * 5000` synthetic probe ŌĆö directly inflates the
  record without touching the blob. Story body could be tightened in
  a future doc-only pass to say "an oversized field that lands in the
  record (e.g., a pathological version string)."
- **`mkdir(0o700)` on macOS respects umask.** `Path.mkdir(parents=True)`
  with no explicit `mode=` falls back to the current process umask, which
  on macOS dev shells is `0o022` ŌåÆ directories come back as `0o755`.
  `_ensure_dir` calls `os.chmod(path, _DIR_MODE)` immediately after
  `mkdir` to defeat umask. The same fix is needed in the blob shard
  directory creation path; the test
  `test_post_write_modes_pin_dirs_blobs_and_index` is the load-bearing
  assertion that catches this if a future refactor drops the chmod.
- **`os.replace` raising leaves a `.tmp` file behind.** The atomic-write
  test asserts no `*.json` file is visible at the final path ŌĆö but the
  `.tmp` file does remain after the mocked failure. This is consistent
  with the invariant ("partial file is never visible at the final path")
  and matches the architecture; `cache gc` (S4-02 stub) would sweep
  orphaned `.tmp` files. Worth noting for Phase 1+ if a `gc` body is
  written.
