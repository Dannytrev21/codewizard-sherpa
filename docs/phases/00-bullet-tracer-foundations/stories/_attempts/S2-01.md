# S2-01 — Error hierarchy + structlog config — attempt log

## Attempt 1 — 2026-05-13 — green

**Outcome:** Done. All 7 ACs verified; 94/94 suite green (one pre-existing
coverage warning unchanged); ruff, mypy --strict, pre-commit all clean.

### What worked

- Single TDD red-marker confirmed both modules absent (`ModuleNotFoundError`
  on import); single green pass made all 15 tests pass with one exception
  (see "What didn't, the first time" below).
- Caching `structlog.configure(**kwargs)` by `(id(sys.stderr), is_tty, verbose)`
  satisfied AC-7's idempotency requirement cleanly: same triple → same dict
  → identical `structlog.get_config()` mapping; different triple →
  rebuild & re-apply, so verbose flips work.
- `structlog.PrintLoggerFactory(file=sys.stderr)` constructed inside
  `_build_config` (not at module load) lets `capsys` capture output — its
  replacement of `sys.stderr` happens before our call.

### What didn't, the first time

- AC-4 mutation guard `set(cls.__dict__.keys()) <= {"__module__",
  "__qualname__", "__doc__"}` red-on-first-run: Python 3.13 auto-injects
  `__firstlineno__` and `__static_attributes__` into every class
  `__dict__`. The AC's set was written assuming pre-3.13. Resolution:
  widen the allowed set in the test to include both compiler-injected
  keys (annotated inline). Intent preserved.
- Full `pytest` run after green-phase: `test_lint_imports_canary` failed
  because import-linter's "codegenie (__init__) must not top-level import
  heavy modules" contract scoped `source_modules = ["codegenie"]` as the
  whole package, treating the new `codegenie.logging -> structlog` edge
  as a violation. The documented contract name pins scope to
  `__init__.py`. Resolution: add `as_packages = false` to that contract;
  both halves of the S1-05 AC-10 canary still pass.

### Lesson for the next story

- Python 3.13 silently grew class `__dict__` — any story that pins a
  closure on `__dict__.keys()` must allow `__firstlineno__` and
  `__static_attributes__`. Promoted to `_lessons.md`.
- import-linter's `as_packages` flag distinguishes "this package's
  `__init__.py`" from "this package and every descendant" — story authors
  who want cold-start coverage of just the package init must opt out of
  the default. Promoted to `_lessons.md`.
