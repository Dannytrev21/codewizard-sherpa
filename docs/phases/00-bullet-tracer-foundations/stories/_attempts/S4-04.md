### Follow-ups surfaced this attempt (not folded in)

- **Sanitizer scope widening.** The current `OutputSanitizer.scrub` only operates on `ProbeOutput.schema_slice`. The envelope's `repo.root` and any future top-level path fields are scrub-naked. Phase 1 should either widen the sanitizer to be a true `RepoContext` chokepoint or ADR-amend to declare envelope-level paths the caller's responsibility (the current `repo.root` basename redaction is the latter, hard-coded in the CLI).
- **Probe-internal vs coordinator-emitted `probe.success` event name collision.** `LanguageDetectionProbe` (S4-01) emits its own `probe.success` with `count_total`/`confidence`; the coordinator (S3-05) also emits one with `duration_ms`/`cache_key`. Smoke tests disambiguate by `cache_key in event`. Phase 6's state ledger will need an explicit rule for which one is canonical. Consider renaming the probe-internal event (e.g., `probe.success.body` or just removing it ‚Äî the coordinator's is already a strict superset).
- **`audit verify` output sentinel.** The command exits 0/4 but prints nothing on success. A `0 mismatch(es)` (or similar) sentinel on stdout would make CI log diagnostics tighter. Not blocking ‚Äî exit-code gating is sufficient ‚Äî but a small QoL win.
