### Decisions & deviations from the story text

1. **`probes` is `Sequence[Probe]` (instances), not `list[type[Probe]]`.** AC-3 wording said `list[type[Probe]]` but every test in the TDD plan passes instances (`FakeProbe(name="p1")` with per-instance `_run` / `_seen_snapshots` hooks; `probe.run = AsyncMock(...)` for cache-hit tests). Followed the tests, the more concrete contract. Documented here as a follow-up doc-lag.
2. **`BudgetingContext` is NOT a `ProbeContext` subclass.** AC-20's literal constructor signature is `BudgetingContext(workspace: Path, raw_artifact_mb: int)` — two fields only. The coordinator passes `BudgetingContext` as the `ctx` argument to `probe.run` (duck-typed; `Probe.run` signature types it as `ProbeContext` but Python doesn't enforce). Phase-0 probes (only `LanguageDetectionProbe` in S4-01) don't read `cache_dir`/`output_dir`/`logger`/`config`. Phase 1+ probe-authoring guide will codify which fields are required.
3. **AC-8 `KeyboardInterrupt` test uses a custom `BaseException` subclass.** Pytest's runner intercepts `raise KeyboardInterrupt()` regardless of `pytest.raises` / `try`-context (pytest treats it as Ctrl-C abort). A project-local `_BaseExcCarveOut(BaseException)` exercises the SAME `except Exception` carve-out the story names without colliding with pytest's abort path. Documented in the test.
4. **`cache.put` is skipped on the failure path.** The implementer outline reads `cache.put` after step 8 (sanitize); applied only to clean successes (`if not sanitized.errors:`). Caching failures would persist a `confidence="low"` result across retries — bad UX. The cache-hit path still works because cached entries are by definition successes.
5. **`_format_secret_error` defensive unwrap.** Strict mypy refused `tuple[()]` indexing of `getattr(...args, ())`; I cast via `tuple(getattr(inner, "args", ()))` first so the slot indexing is over a `tuple[Any, ...]`. Function still produces the exact regex shape pinned in AC-13: `SecretLikelyFieldNameError: <key> at (<path>,)`.
6. **`os.cpu_count()` darwin branch is via a divisor variable, not `if sys.platform == ...: return`.** Mypy strict + `warn_unreachable` flags `return` after a `sys.platform == "darwin"` branch as dead code on the dev platform. The single-expression `divisor = ... if ... else ...` is portable and strict-clean.
7. **`structlog.contextvars.bind_contextvars` + `clear_contextvars` in `finally`.** Implementer note 11 — without the finally clause, subsequent tests in the same process inherit the binding and per-test event-stream isolation breaks. The test suite passes deterministically across multiple runs as a result.
8. **`_isolated_snapshot` copies `detected_languages` and `config` dicts.** AC-18's mutation-leak test would fail if we only ran `dataclasses.replace(snap)` — the dict fields are shared references. Explicit shallow-copy of the two mutable dicts is the runtime defense.

## Lessons (appended to `_lessons.md`)

- **Pytest 9 intercepts `raise KeyboardInterrupt()` regardless of `pytest.raises` / in-test `try` context** — it short-circuits to "user pressed Ctrl-C" and aborts the runner before the except clause executes. Use a project-local `BaseException` subclass for AC-style "BaseException carve-out" coverage.
- **`mypy --strict` + `warn_unreachable` flags `sys.platform == "darwin"`-gated returns as dead code** on the dev platform. Compute a divisor variable with a single ternary instead of branching with `return` statements.
- **`structlog.contextvars.bind_contextvars` MUST be paired with `clear_contextvars()` in `finally`.** Otherwise per-test event-stream isolation breaks when tests share the same event loop or process — `run_id` bleeds across tests.
- **`dataclasses.replace(snap)` does NOT isolate mutable fields.** It produces a fresh outer dataclass instance but the dict/list fields are still shared references. To defend against in-place mutation (AC-18), shallow-copy the mutable fields explicitly.
- **`Probe.run` signature types `ctx: ProbeContext` but Python doesn't enforce it.** The coordinator can pass a `BudgetingContext` (or any duck-typed surface) provided downstream probes only call attributes/methods the duck-typed object exposes. Phase 1 probe-authoring guide should codify which `ProbeContext` attributes/methods are MANDATORY vs OPTIONAL.
- **Story `probes: list[type[Probe]]` vs test instances** — when an AC's signature contradicts the concrete test code, follow the tests. AC text is doc-lag; tests are the runtime contract. Surface as a follow-up.
- **`tests/unit/test_errors.py` pins the public surface of `codegenie.errors`.** Adding a new typed error class (`ProbeBudgetExceeded` here) requires updating `EXPECTED_SUBCLASSES` AND adding a docstring containing one of the slugs in `DOCUMENTED_MODULE_SLUGS` ({exec, cache, sanitizer, validator, writer, coordinator, config, tool_check, schema}). Forgetting either fails CI.
