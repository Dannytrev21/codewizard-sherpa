### Surprises

- **`Probe.layer` / `Probe.tier` accept bare `Literal`-typed strings without
  `# type: ignore`.** I initially added `# type: ignore[assignment]` comments
  on the class attribute declarations expecting mypy to complain about
  `Literal["A".."G"]` narrowing — mypy 1.x is smart enough to accept the
  literal string here. The unused-ignore comments tripped `[unused-ignore]`
  in pre-commit; removed and clean.
- **`PermissionError IS OSError`** — the implementation outline's "wrap for
  `PermissionError` and other `OSError` subclasses" reads as two separate
  except clauses, but a single `except OSError` captures both and produces
  `"PermissionError: ..."` via `type(exc).__name__` for the AC-5 assertion.
  Collapsed to one clause per CLAUDE.md Rule 2 (Simplicity First).

  prepending `.venv/bin` to `PATH` when invoking pytest. Tests pass under
  that PATH.
