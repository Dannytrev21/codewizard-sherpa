### What worked

- **The validator's hardened skeleton paid off.** AC-4c's six-arg
  `coordinator.gather(...)` dispatch landed in one pass because the story
  already named the signature shape; the only adjustment was binding
  `Task(type=..., options={})` (per the actual frozen ABC) instead of the
  skeleton's `Task(name=..., languages=...)` — see Lesson 1 below.
- **AC-4d's monkeypatch path** (`codegenie.coordinator.validator._ProbeOutputValidator.model_validate`)
  works through the coordinator's lazy import (`coordinator.py:320`) because
  `monkeypatch.setattr` mutates the class attribute on the module the lazy
  import resolves to. No additional hooks needed.
- **Symlink-event payload schema** matched the story exactly:
  `escaped[0]["path"] == "link.js"`, no `/etc/hosts` in the str of the
  event, closed-world `{"javascript": 1}` count.

### What didn't (initial RED)

- **First run of `test_symlink_out_of_repo_skipped` failed** with
  `KeyError: 'language_stack'`. Cause: the story's AC-3 said "load the
  YAML, assert `data['language_stack']['counts']`" — but the envelope
  shallow-merges probe outputs under `envelope['probes'][<probe_name>]`,
  so the actual path is `data['probes']['language_detection']['language_stack']['counts']`.
  Fixed by reading `data["probes"]["language_detection"]["language_stack"]["counts"]`.
  See Lesson 2 below.
- **`lint-imports` was missing from PATH** on first full-suite run; this is
  a pre-existing dev-env issue (S1-05 ships it via `[dev]`). `pip install
  import-linter` + `PATH="$PWD/.venv/bin:$PATH"` cleared it. Not caused by
  this story.

### Cross-story lessons appended

See `_lessons.md` for the two new entries discovered in this attempt:

1. **`Task` is `(type=..., options={})`, not `(name=..., languages=...)`** —
   Story skeletons can lag the frozen ABC; always reconcile against
   `src/codegenie/probes/base.py` at implementation time.
2. **The YAML envelope nests probe outputs under `probes[<name>]`** —
   `_seam_shallow_merge` (cli.py:260) places each `SanitizedProbeOutput.schema_slice`
   under `envelope["probes"][probe.name]`. Tests asserting on `language_stack`
   in the rendered YAML must read `data["probes"]["language_detection"]
   ["language_stack"]`, not `data["language_stack"]`.
