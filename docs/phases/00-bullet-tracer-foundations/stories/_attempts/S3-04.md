### What landed

- `src/codegenie/config/__init__.py` — re-exports `Config`, `load_config`; `__all__ = ["Config", "load_config"]`.
- `src/codegenie/config/defaults.py` — frozen `@dataclass Config` (three fields pinned), `_defaults()`, `_known_fields()`. Stdlib-only at module top level (cold-start budget).
- `src/codegenie/config/loader.py` — four-source merge:
  - `_read_yaml_if_exists` returns `{}` on `FileNotFoundError`; wraps `yaml.YAMLError` into `ConfigError` with `from err`; rejects non-mapping top level via `isinstance(parsed, Mapping)`.
  - `_check_unknown_keys` formats `"unknown key '<k>'; did you mean '<closest>'?"`, omits the clause when `difflib.get_close_matches(cutoff=0.6)` returns `[]`.
  - `_typed_construct` uses `typing.get_type_hints(Config)` for per-field `isinstance` enforcement (dataclasses don't type-check at runtime), wraps the resulting `TypeError` via `raise ConfigError(...) from err`.
  - `_provenance` walks declared fields highest-to-lowest source, labels from the closed set `{"defaults", "global", "repo", "cli"}` matching `final-design.md §2.13`.
  - `load_config` validates keys per source *before* merging, dict-merges in precedence order, emits one `config.loaded` DEBUG event with `provenance=...`, then constructs.
  - `structlog` import is function-local; `defaults.py` and `loader.py` import zero third-party heavy modules at top level (`yaml` is the only third-party top-level import in `loader.py`).
- `tests/unit/test_config_loader.py` — 34 tests across the seven AC sections, fixtures `write_user_yaml` / `write_repo_yaml` monkeypatch `Path.home` so the developer's `~/.codegenie/` is never touched.

### Decisions worth noting

1. **Per-field `isinstance` enforcement via `typing.get_type_hints(Config)`.** The story implementer-note flagged that bare `Config(**merged)` does not raise `TypeError` on a string-where-int value (dataclasses don't validate types). The cleanest typed approach is `get_type_hints` (stdlib), which resolves the `from __future__ import annotations` string forms to real types. Lazy-resolves once per `_typed_construct` call. Zero `# type: ignore` in new src code (AC-17).
2. **`Mapping` (collections.abc) for the non-mapping check, not `dict`.** Matches the story note (line 658): looser, future-proof against PyYAML returning subtypes; `dict(parsed)` immediately after to give the rest of the loader a plain dict.
3. **`del source_name` in `_check_unknown_keys`.** Kept the parameter (it documents the call sites — global/repo/cli) but discarded it because AC-11's pinned message format does not echo the source name. Avoids `unused-argument` lint and `Vulture`-style flags later.
4. **Lint-imports canary failure is environment-only and pre-existing.** `test_lint_imports_exits_zero_on_current_tree` fails locally because `lint-imports` is in `.venv/bin/` but not on system `PATH`. Verified by running `.venv/bin/lint-imports --config pyproject.toml --no-cache` directly: both contracts KEPT. Same two tests fail on master without my changes — confirmed by running them with `S3-04` files removed-from-stash. Not blocking.

### Lessons (carry-forward — appended to `_lessons.md`)

- `typing.get_type_hints(<dataclass>)` is the simplest way to resolve PEP 563 (`from __future__ import annotations`) string forms to actual types when a dataclass needs per-field `isinstance` checks at runtime. Plain `field.type` returns the unresolved string under `__future__ annotations`. Stdlib-only — no `inspect` or `eval` needed.
- The story's `_typed_construct` spy idiom (AC-14) requires the helper to live at module scope as a named function the test can `monkeypatch.setattr`. A nested closure or a method would not be patchable. This shaped the loader's helper layout: every wrappable surface is a module-scope `_underscored` function.
