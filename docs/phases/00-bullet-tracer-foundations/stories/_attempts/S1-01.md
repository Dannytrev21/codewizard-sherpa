### Decisions / deviations

- **`[tool.hatch.version].pattern` override.** Hatchling's default regex (`^__version__\s*=\s*['"](?P<version>...)['"]`) does not tolerate the PEP 526 type annotation that the story's Refactor section asks for (`__version__: str = "0.1.0"`). Rather than drop the annotation, I added a one-line `pattern = "..."` override that accepts an optional `: <type>` between the name and the `=`. Reversible; documented inline in `pyproject.toml`. No AC modified.
- **`__main__.py` doesn't import from `codegenie.cli`.** Implementation outline §4 hinted "imports a `main` callable from `codegenie.cli` *lazily*", but "Out of scope" explicitly defers `src/codegenie/cli.py` to S4-02. To resolve the conflict, the placeholder click command lives directly in `__main__.py`. When S4-02 introduces `cli.py`, `__main__.py` can be rewritten to delegate without breaking AC-8.
- **`README.md` added.** Not in the story's "Files to touch", but `[project].readme = "README.md"` is required by PEP 621 for the wheel build to succeed cleanly. One-paragraph pointer to `docs/` only — no new content.

### Lessons (also flagged in `_lessons.md`)

- Hatchling's default version regex collides with PEP 526 type annotations on `__version__`. Stories that ask for a `__version__: str` constant under hatchling's AST hook must either drop the annotation or override `pattern`.
- Click's standalone-mode dispatch raises `click.exceptions.Exit` on `--help`/`--version`; converting it to a return code in `main()` keeps the `python -m` entry test-friendly without sacrificing click's stdout output.
