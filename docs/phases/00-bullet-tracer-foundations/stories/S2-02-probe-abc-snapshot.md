## Context

This is **the load-bearing story of the entire phase**. It transcribes `localv2.md §4`'s `Probe` ABC and four dataclasses byte-for-byte into `src/codegenie/probes/base.py`, then plants the snapshot test that pins the contract in CI. Once this lands, the ABC is frozen: every later phase (1–14) inherits the same `Probe` shape, and any drift — code-side or doc-side — fails CI with a message routing the offender to `templates/adr-amendment.md`. A transcription error encoded on day one becomes a permanent error inherited by every probe ever written. The regen script, not your retyping, is the source of truth.

Foundational work: nothing else in Step 2 (and almost nothing in Phase 1+) can compile without `probes/base.py`.

## Goal

`tests/unit/test_probe_contract.py` runs the in-repo regen script against `localv2.md §4` and the implemented `Probe` class, compares both artifacts to `tests/snapshots/probe_contract.v1.json`, and passes; a single-character edit to `localv2.md §4` makes it fail with a message pointing at `templates/adr-amendment.md`.

## Acceptance criteria

- [ ] `src/codegenie/probes/base.py` contains the `Probe` ABC plus `RepoSnapshot`, `Task`, `ProbeContext`, `ProbeOutput` dataclasses byte-for-byte from `localv2.md §4` (no field reordering, no type-hint substitutions, no docstring rewording).
- [ ] `scripts/regen_probe_contract_snapshot.py` extracts `§4` from `localv2.md` by section anchors (`## 4. ...` start, next top-level anchor end), normalizes it (whitespace-collapsed, no trailing newlines), and writes `tests/snapshots/probe_contract.v1.json` containing `{ "doc_fingerprint": "<sha256-hex>", "structural_signature": {...} }` per ADR-0007.
- [ ] `tests/snapshots/probe_contract.v1.json` is committed with the regen output for the §4 body at the close of this story.
- [ ] `tests/unit/test_probe_contract.py` re-runs the regen logic and asserts both `doc_fingerprint` and `structural_signature` match the committed snapshot byte-for-byte; failure mode message contains the path `templates/adr-amendment.md` per ADR-0007 enforcement-loop policy.
- [ ] Manually editing a non-whitespace character in `localv2.md §4` (verified by a synthetic test that monkeypatches the loader) causes `test_probe_contract.py` to fail with the documented message.
- [ ] `templates/adr-amendment.md` exists with a checklist routing the contributor through (a) edit `localv2.md`, (b) run the regen script, (c) update implementation if structural drift surfaces, (d) reference ADR-0007 in the PR.
- [ ] `ruff check`, `ruff format --check`, `mypy --strict`, and `pytest` all pass on the touched files.

## Implementation outline

1. Write `scripts/regen_probe_contract_snapshot.py` **first** — programmatic extraction of `localv2.md §4` by section anchors, normalization (`re.sub(r"\s+", " ", body).strip()`), SHA-256-hex of the normalized bytes, plus a structural-signature builder that walks `codegenie.probes.base` and serializes class names, MRO, field names, type annotations, defaults.
2. Run the regen script from a fresh extraction of `localv2.md §4` — capture the doc fingerprint **before** writing `base.py`. This locks the "doc is source of truth" direction.
3. Implement `src/codegenie/probes/base.py` by literal copy from `localv2.md §4` (use an editor's clipboard from the rendered doc, then diff). Resolve any whitespace-only differences in favor of the doc; structural drift fails the regen.
4. Run the regen script again to populate `structural_signature`; commit `tests/snapshots/probe_contract.v1.json`.
5. Write `tests/unit/test_probe_contract.py` that imports the regen helpers from `scripts.regen_probe_contract_snapshot` (or factor a shared `_normalize_and_hash` helper into a tested module so the test doesn't shell out).
6. Write `templates/adr-amendment.md` with the four-step checklist.
7. Manually diff `src/codegenie/probes/base.py` against `localv2.md §4` before opening the PR (per Implementation-level risk #1).

## TDD plan — red / green / refactor

### Red — write the failing test first

Test file path: `tests/unit/test_probe_contract.py`.

This story has **two** anchoring tests because the snapshot has two artifacts (the doc fingerprint *and* the structural signature) and they fail for different reasons.

```python
# tests/unit/test_probe_contract.py
import json
from pathlib import Path
import codegenie.probes.base as base
from scripts.regen_probe_contract_snapshot import (
    extract_section_4_body, normalize_and_hash, structural_signature,
)

SNAPSHOT_PATH = Path(__file__).resolve().parents[1] / "snapshots" / "probe_contract.v1.json"
LOCALV2_PATH = Path(__file__).resolve().parents[2] / "docs" / "localv2.md"

def test_probe_contract_doc_fingerprint_matches_snapshot():
    # arrange: load committed snapshot + current localv2 §4 body
    snap = json.loads(SNAPSHOT_PATH.read_text())
    body = extract_section_4_body(LOCALV2_PATH.read_text())
    # act: recompute fingerprint
    current = normalize_and_hash(body)
    # assert: byte-for-byte equality; failure message routes to amendment template
    assert current == snap["doc_fingerprint"], (
        "localv2.md §4 has drifted from the frozen contract. "
        "Resolution is ALWAYS 'change code to match doc, never the inverse'. "
        "See templates/adr-amendment.md and ADR-0007."
    )

def test_probe_class_structural_signature_matches_snapshot():
    # arrange: same snapshot
    snap = json.loads(SNAPSHOT_PATH.read_text())
    # act: rebuild structural signature from the actual Probe class + dataclasses
    current = structural_signature(base)
    # assert: structural drift fails; same failure routing applies
    assert current == snap["structural_signature"], (
        "src/codegenie/probes/base.py has drifted from localv2.md §4. "
        "See templates/adr-amendment.md and ADR-0007."
    )
```

A *third* synthetic test confirms the failure-message-pointer policy works on doc drift (monkeypatch the loader to return a tampered body and assert the failure message contains `templates/adr-amendment.md`). Aim for one assertion per behavior; don't multiply.

Confirm all three fail with `ImportError` (no `base` module, no regen helpers). Commit the red marker.

### Green — make it pass

`scripts/regen_probe_contract_snapshot.py`: define `extract_section_4_body(md_text: str) -> str` slicing between `\n## 4.` and the next `\n## ` anchor; `normalize_and_hash(body: str) -> str` collapses runs of whitespace to single spaces, strips, returns `hashlib.sha256(body.encode("utf-8")).hexdigest()`; `structural_signature(module) -> dict` returns a sorted dict of `{class_name: {"bases": [...], "fields": [{"name": ..., "type": ..., "default": ...}, ...]}}` using `dataclasses.fields()` and `inspect.get_annotations()`.

Run `scripts/regen_probe_contract_snapshot.py`; commit the resulting JSON file.

## Notes for the implementer

- **Implementation-level risk #1 from `High-level-impl.md` is this story.** A transcription error is the snapshot. Write the regen script first, run it from a fresh `git pull` of `localv2.md`, *then* write `base.py`. Manually diff `base.py` against `localv2.md §4` line-by-line before opening the PR.
- The normalization algorithm is fixed: `re.sub(r"\s+", " ", body).strip()`. Resist "improvements" like stripping comments — the algorithm must be auditable from the script itself, and `phase-arch-design.md §Open questions Q3` documents this exact normalization.
- The structural signature must not include attribute *order* if `__annotations__` ordering varies across Python versions — but in 3.11+ it's insertion-stable, so order-preserving serialization is the safer default. Test on both 3.11 and 3.12 (CI matrix covers it).
- Per `phase-arch-design.md §Edge cases` row 10, the failure-message-points-at-`adr-amendment.md` text is part of the contract. Do not paraphrase it — copy the exact pointer string into the test failure message.
- The snapshot filename has `v1` baked in (`probe_contract.v1.json`). A future v2 bump *creates* `probe_contract.v2.json` rather than overwriting v1 — old snapshot preserved for historical replay (ADR-0007 §Consequences).
- `scripts/` is a top-level directory peer to `src/` and `tests/`; ensure `pyproject.toml` doesn't treat it as a package (no `__init__.py`). The test imports from `scripts.regen_probe_contract_snapshot` — confirm `pyproject.toml`'s `[tool.pytest.ini_options].pythonpath` (set in S1-02) includes `.` so the bare-script import works, or factor the helpers into a tested module under `src/`.
- `CODEOWNERS` will gate `src/codegenie/probes/base.py`, `localv2.md`, and `tests/snapshots/` in S5-02 — this story doesn't need to add the entry, but the snapshot's *political* immutability per ADR-0007 §Reversibility depends on that gate landing in Step 5.
