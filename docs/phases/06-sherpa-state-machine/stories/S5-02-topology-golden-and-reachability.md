# S5-02 — Topology golden file + reachability tests

**Step:** Step 5 — Implement `build_vuln_loop()` lazy-singleton factory + topology golden + `interrupt_before`
**Effort:** M
**Depends on:** S5-01 (factory exists; compile succeeds; `interrupt_before` wired)
**Status:** Backlog
**ADRs honored:** ADR-0013 (JSON-form golden topology snapshot is the CI gate; SVG is review-only), ADR-0001 (factory shape — golden test uses `force_rebuild=True` + `InMemorySaver()` to keep the test hermetic)

## Context

The factory compiles, but nothing yet catches **unintended topology changes**. A future PR that adds a stray edge, renames a node, or flips a conditional-label key would slip past every existing test as long as the predicate-level unit tests still pass. The topology is small, deterministic, and central — exactly the surface area where a golden-file gate is cheapest and highest value (arch §Component 10, ADR-0013).

Two non-trivial choices, both from ADR-0013:

1. **JSON, not SVG.** `critique.md best-practices-hidden-1` killed the SVG-as-gate idea: `langgraph-cli` SVG output embeds layout coordinates and version-sensitive metadata. A Renovate bump of `langgraph-cli` would flip the gate red on every minor version. `graph.get_graph().to_json()` is the documented LangGraph API surface; its key set is far more likely to be stable. The JSON form is the CI gate. The SVG is committed for human review at `docs/phases/06-sherpa-state-machine/vuln_loop.svg` but does **not** fail CI on drift. (S6-04 ships the `codegenie loop render` command that produces both.)
2. **Updating the golden requires an explicit `--update-golden` flag.** Without that friction, a sloppy PR re-snapshots the golden alongside an unintentional topology change and the gate becomes a rubber-stamp. The `--update-golden` flag forces the author to look at the diff and justify it in PR review.

This story ships three things:

- The canonicalization helper (`canonical_json(value) -> bytes`) that recursively sorts dict keys, normalizes a small whitelist of version-sensitive fields, and serializes with `separators=(",", ":")` + UTF-8 + no trailing newline.
- The golden file itself, `tests/golden/vuln_loop_topology.json`, generated by running the test once with `--update-golden`.
- Two tests: a byte-diff CI gate (`test_topology_golden.py`) and a reachability proof (`test_topology_reachability.py`) that every node is reachable from `ingest_cve` and both `END`-terminal paths (`emit_artifact → END` and `escalate → END`) are reachable from at least one conditional-edge label.

## Goal (one sentence)

Ship `tests/golden/vuln_loop_topology.json` (the canonicalized `build_vuln_loop().get_graph().to_json()` form), a byte-diff CI gate (`tests/graph/test_topology_golden.py`) updatable only via an explicit `--update-golden` flag, and a reachability test (`tests/graph/test_topology_reachability.py`) proving every node is reachable from `ingest_cve` and both terminal paths reach `END`.

## Acceptance criteria

- [ ] `tests/golden/vuln_loop_topology.json` exists, is canonical-JSON-encoded (recursively sorted keys, `separators=(",", ":")`, UTF-8, no trailing newline), and was generated by running the test once with `--update-golden`.
- [ ] `tests/graph/test_topology_golden.py` reads the golden, computes `canonical_json(build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True).get_graph().to_json())`, and `assert actual == expected` on the raw bytes. The diff is shown in test output on failure (use `assert ... == ...` against a `bytes` literal so pytest renders a side-by-side, or convert both to `str` for the assertion and back-encode for write).
- [ ] **Canonicalization is documented and minimal.** `canonical_json(value: object) -> bytes` recursively sorts dict keys and strips a small **named, documented** whitelist of fields that are not load-bearing for topology (e.g., LangGraph-injected timestamp/version metadata if present). The whitelist lives in `src/codegenie/graph/_golden.py` (or `tests/golden/_canonical.py`) as a `_STRIPPED_KEYS: frozenset[str]` constant; `tests/golden/README.md` lists each stripped key and why. **No silent stripping.**
- [ ] `tests/graph/test_topology_reachability.py` proves: (a) every node in `{ingest_cve, select_recipe, apply_recipe, rag_lookup, replan_with_phase4, validate_in_sandbox, record_attempt, await_human, emit_artifact, escalate}` is reachable from `ingest_cve` via the directed edges, (b) both `emit_artifact → END` and `escalate → END` are reachable, (c) **every** conditional-edge label (the 9 labels across the 4 conditional sites — `"matched"`, `"miss"` on `select_recipe`; `"hit"`, `"miss"` on `rag_lookup`; `"passed"`, `"retry_phase4"`, `"retry_exhausted"`, `"non_retryable"` on `record_attempt`; `"continue"`, `"override"`, `"abort"` on `await_human`) has a downstream path to `END`.
- [ ] **`--update-golden` flag** is implemented via a pytest CLI plugin (`tests/conftest.py` adds `--update-golden` to the option parser); when set, the golden test writes the canonicalized actual bytes to disk and skips the byte-diff. When unset (default), it asserts. Running `pytest tests/graph/test_topology_golden.py --update-golden` is the only sanctioned way to bump the golden.
- [ ] **Red test exists and was committed before the implementation** (see TDD plan); it now passes.
- [ ] `ruff format`, `ruff check`, `mypy --strict src/codegenie/graph/ tests/graph/`, and `pytest tests/graph/` are clean.

## Implementation outline

1. **Implement `canonical_json` in a single place.** Best location: `src/codegenie/graph/_golden.py` (private module, leading underscore — internal-only contract). The function takes an arbitrary `object` (whatever `get_graph().to_json()` returns — likely a `dict[str, Any]` with nested lists/dicts/strings/ints), recursively sorts dict keys, strips the documented whitelist of version-sensitive keys, and serializes via `json.dumps(..., separators=(",", ":"), ensure_ascii=False, sort_keys=True).encode("utf-8")`. No trailing newline. `sort_keys=True` is the belt-and-suspenders behind the recursive sort.
2. **Probe `to_json()` once, record the stripped-key whitelist.** Run the factory at the REPL: `build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True).get_graph().to_json()`. Inspect the output for any field that's likely to drift on LangGraph minor bumps (timestamps, version strings, internal object ids). For each, add to `_STRIPPED_KEYS` with a one-line rationale in `tests/golden/README.md`. Err on the side of **stripping less** — if you're not certain a field is version-sensitive, leave it in; the gate will catch the drift later and the next PR can promote it to the whitelist.
3. **Add the `--update-golden` flag.** In `tests/conftest.py`:
   ```python
   def pytest_addoption(parser):
       parser.addoption(
           "--update-golden",
           action="store_true",
           default=False,
           help="Rewrite golden files with the actual test output. "
                "Use deliberately; the diff appears in PR review.",
       )

   @pytest.fixture
   def update_golden(request) -> bool:
       return request.config.getoption("--update-golden")
   ```
4. **Generate the golden file.** Once the implementation is in place, run `pytest tests/graph/test_topology_golden.py --update-golden` exactly once. Commit the produced `tests/golden/vuln_loop_topology.json` in the same PR. Also commit `tests/golden/README.md` documenting the canonicalization rules and the `--update-golden` workflow.
5. **Write the reachability test.** Walk the graph from `ingest_cve` via `g.edges` (LangGraph exposes the edge set on the inspected graph), collect reachable nodes, assert the set equals the ten expected. For conditional-edge label coverage, iterate the labels on each conditional-edge mapping (from arch §Component 2) and assert each downstream node is in the reachable set and has a path to `END`. Use a small BFS — no external library; `collections.deque` is sufficient.
6. **Do not commit the SVG.** That's S6-04's job (`codegenie loop render`).

```python
# tests/graph/test_topology_golden.py
from pathlib import Path
import pytest
from langgraph.checkpoint.memory import InMemorySaver
from codegenie.graph import build_vuln_loop
from codegenie.graph._golden import canonical_json

GOLDEN = Path(__file__).resolve().parents[1] / "golden" / "vuln_loop_topology.json"


def test_topology_golden_matches(update_golden):
    """The compiled graph's JSON form matches the committed golden, byte-for-byte.

    Why this matters: a stray edge or a renamed node would otherwise slip past
    every predicate-level unit test as long as the predicates still return the
    same labels. The golden gate is the single CI signal that the topology
    didn't change unintentionally.
    """
    compiled = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True)
    actual = canonical_json(compiled.get_graph().to_json())
    if update_golden:
        GOLDEN.parent.mkdir(parents=True, exist_ok=True)
        GOLDEN.write_bytes(actual)
        pytest.skip("golden updated; re-run without --update-golden to verify")
    expected = GOLDEN.read_bytes()
    assert actual == expected, (
        "Topology drift detected. If intentional, re-run with --update-golden "
        "and explain the diff in the PR description."
    )


def test_canonical_json_is_byte_stable_across_invocations():
    """canonical_json is deterministic across runs — same input, same bytes."""
    compiled = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True)
    raw = compiled.get_graph().to_json()
    a = canonical_json(raw)
    b = canonical_json(raw)
    assert a == b
    assert isinstance(a, bytes)
    assert not a.endswith(b"\n"), "canonical_json must not append a trailing newline"


def test_canonical_json_sorts_nested_keys():
    """Verify recursive key-sort, not just top-level."""
    from codegenie.graph._golden import canonical_json
    a = canonical_json({"b": {"y": 1, "x": 2}, "a": [{"z": 3, "k": 4}]})
    b = canonical_json({"a": [{"k": 4, "z": 3}], "b": {"x": 2, "y": 1}})
    assert a == b


# tests/graph/test_topology_reachability.py
from collections import deque
from langgraph.checkpoint.memory import InMemorySaver
from codegenie.graph import build_vuln_loop

EXPECTED_NODES = {
    "ingest_cve", "select_recipe", "apply_recipe", "rag_lookup",
    "replan_with_phase4", "validate_in_sandbox", "record_attempt",
    "await_human", "emit_artifact", "escalate",
}


def _bfs(adj: dict[str, set[str]], start: str) -> set[str]:
    seen, q = {start}, deque([start])
    while q:
        n = q.popleft()
        for m in adj.get(n, set()):
            if m not in seen:
                seen.add(m)
                q.append(m)
    return seen


def test_every_node_reachable_from_ingest_cve():
    compiled = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True)
    g = compiled.get_graph()
    adj: dict[str, set[str]] = {}
    for e in g.edges:
        adj.setdefault(e.source, set()).add(e.target)
    reachable = _bfs(adj, "ingest_cve")
    missing = EXPECTED_NODES - reachable
    assert not missing, f"unreachable from ingest_cve: {missing}"


def test_both_terminal_paths_reach_end():
    compiled = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True)
    g = compiled.get_graph()
    targets_from = {n: {e.target for e in g.edges if e.source == n} for n in EXPECTED_NODES}
    assert "__end__" in targets_from["emit_artifact"]
    assert "__end__" in targets_from["escalate"]


def test_every_conditional_label_reaches_end():
    """Every conditional-edge label has at least one downstream path to END.

    Why this matters: a conditional-edge label whose target accidentally
    became orphaned (e.g., a refactor that deleted the node it pointed to but
    left the label in the mapping) is exactly the bug the predicate-level
    tests would miss.
    """
    compiled = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True)
    g = compiled.get_graph()
    adj: dict[str, set[str]] = {}
    for e in g.edges:
        adj.setdefault(e.source, set()).add(e.target)
    # Targets that conditional edges feed into per arch §Component 2.
    for downstream in {
        "apply_recipe", "rag_lookup", "replan_with_phase4",
        "emit_artifact", "await_human", "escalate",
    }:
        reachable = _bfs(adj, downstream)
        assert "__end__" in reachable, f"{downstream} has no path to END"
```

## Notes for the implementer

- **Look at the raw `to_json()` output before committing the whitelist.** A reasonable smoke test:
  ```python
  import json
  from langgraph.checkpoint.memory import InMemorySaver
  from codegenie.graph import build_vuln_loop
  raw = build_vuln_loop(checkpointer=InMemorySaver(), force_rebuild=True).get_graph().to_json()
  print(json.dumps(raw, indent=2, sort_keys=True))
  ```
  If you see keys like `"updated_at"`, `"langgraph_version"`, or any obvious non-topology metadata, add them to `_STRIPPED_KEYS`. If you see only `nodes`, `edges`, `entry_point`, and similar topology-bearing fields — strip nothing.
- **`g.edges` may be a list of tuples or a list of dataclasses, version-dependent.** Probe at the REPL; pin the access pattern (`e.source` vs `e[0]`) for the installed version. Document the version assumption in a one-line comment in the reachability test.
- **`__start__` and `__end__` are LangGraph-internal sentinels** that show up in `g.nodes`. The reachability test filters to `EXPECTED_NODES`; the golden test does **not** filter — internal sentinels are part of the topology contract.
- **Do not let the `--update-golden` flag also rewrite the SVG.** Per ADR-0013, the JSON gate and the SVG renderer are different commands. `--update-golden` is JSON-only.
- **`pytest --update-golden` should be uncommon and called out in PR descriptions.** If the same PR touches `_build()` and the golden, the reviewer wants to see both diffs together; if the same PR touches *only* the golden, that's the smoke. The README is your mechanism for telling future reviewers what to look for.
- **The reachability test should fail loud, not silently.** Avoid generic `assert reachable == EXPECTED_NODES` without showing the missing/extra sets — use the `missing = EXPECTED_NODES - reachable` form and put it in the assertion message. Rule 12 — fail loud.
- **Do not deduplicate the canonicalization helper across `tests/golden/` and `src/codegenie/graph/_golden.py`.** Keep it in `src/` (single source of truth) and have the tests import it. If a test ever monkey-patches the function, that's a bug in the test, not a feature.
- **mypy strict on `tests/conftest.py`** is realistic only if you annotate `pytest_addoption(parser: pytest.Parser) -> None` and the fixture's `request: pytest.FixtureRequest` argument. If LangGraph's `to_json()` returns `Any`, narrow with a `cast(dict[str, Any], raw)` and put a one-line comment citing the LangGraph version — surface the limit of the typing, don't hide it.
