## Executive summary

Phase 2 decomposes into **48 stories** across the 8 steps from [High-level-impl.md](../High-level-impl.md). The distribution is **11 / 7 / 5 / 4 / 4 / 4 / 10 / 3**. Step 1 is the densest — every Phase 2 probe consumes at least one of its primitives (the seven `tools/` wrappers, the sandbox-extension chokepoint, the four ADR-gated Phase 0/1 in-place edits, `OutputSanitizer` Pass 4/5, the audit chain head, the `tools/digests.yaml` pin manifest). Step 2 plants the catalog + skills-loader scaffolding plus three CI lint scripts and the schema-evolution policy. Step 3 lands `IndexHealthProbe` (the load-bearing roadmap exit-criterion probe and the only `consumes_peer_outputs` consumer in Phase 2) and `BuildGraphProbe` with its wrapper-enforced `--ignore-scripts` invariant. Steps 4–7 layer the remaining 15 probes by dependency band (Layer B remainder → Layer C static → Layer C dynamic → Layer D + E + G with the per-file findings sub-cache). Step 8 lands the ≥ 40-fixture adversarial corpus, the 7 integration tests (with the three seeded-staleness fixtures pinning **roadmap exit criterion #2** at "at-least-three" instead of "at-least-one"), the per-probe golden suite, the bench canaries, and the Phase 3 handoff. Cross-step parallelism opens after Step 2 — Layer C static (Step 5) does not block Layer C dynamic (Step 6), and within Step 7 the Layer G probes parallelize with Layer D once the per-file findings cache module is on disk.

## How to use this backlog

1. Start at a story whose dependencies are satisfied (initially, S1-01).
2. Open the story file. Read **Context**, **References**, **Goal**, **Acceptance criteria**.
3. Begin with the **TDD plan — red / green / refactor**. Write the failing test first.
4. Implement just enough to green.
5. Refactor.
6. Check every acceptance criterion. Update Status to `Done`.
7. Move to the next satisfied story.

Order *within* a step is mostly fixed by S-numbering; cross-step parallelism is whatever the DAG allows.

## Definition of done (applies to every story)

- [ ] All acceptance criteria checked.
- [ ] TDD red test exists, committed, green.
- [ ] Additional ADR-honoring tests written and green (e.g. ADR-0001 three-arg dispatch test; ADR-0004 install-time digest test; ADR-0006 sanitizer-idempotence test; ADR-0007 `--ignore-scripts` wrapper-invariant test; ADR-0008 closed-enum parity lint test; ADR-0011 chain-break observability test).
- [ ] Code formatted (`ruff format`), lint clean (`ruff check`), `mypy --strict` on `src/` passes.
- [ ] No existing test disabled or weakened without an explicit note in "Notes for the implementer".
- [ ] Story Status updated to `Done`.
- [ ] If story modifies an ADR-documented contract, that ADR's "Consequences" section reviewed for follow-ups.
- [ ] If story adds a probe, per-module local coverage is reported in the PR body (per "Implementation-level risks" #10 in `High-level-impl.md`); per-module floors at 85/75 for `probes/syft_sbom.py`, `probes/grype_cve.py`, `probes/scip_index.py`; 90/80 for all others.
- [ ] If story adds a sub-schema, the sub-schema declares `schema_version: "v1"`, sets `additionalProperties: false` at every nesting level, and ships an "extra-field rejection" unit test asserting the failing JSON Pointer.

## Cross-cutting concerns

Things that don't belong to any one step but apply across stories.

- **Strict type-checking + lint:** every story passes `ruff check`, `ruff format --check`, and `mypy --strict` on touched `src/` code. Enforced via Definition of Done; no per-story acceptance criterion repeats it.
- **Structured logging via `structlog`:** every story introducing new logged behavior emits one of the Phase-2 event constants registered in S1-01 (`probe.tool.invoked`, `probe.sandbox.network_egress_attempted`, `probe.sanitizer.pass4_fingerprint`, `probe.sanitizer.pass5_marker_detected`, `audit.chain_head.advanced`, `audit.chain_break.detected`, `probe.peer_outputs.snapshot_built`, `index_health.budget_exceeded`) and includes a log-emission assertion in at least one test.
- **Snapshot-frozen contracts:** only S1-11 touches `probes/base.py`; the regen script encodes the allowed attribute list (the only addition is `consumes_peer_outputs`) so a third attribute fails CI ("Implementation-level risks" #3). `CODEOWNERS` gates `probes/base.py` reviews.
- **Chokepoint preservation:** every probe goes through the Phase 0/1 `_ProbeOutputValidator → OutputSanitizer.scrub (passes 1–5) → SchemaValidator → CacheStore.put` chain unchanged. Stories that touch Phase 0/1 code must explicitly justify the edit in "Notes for the implementer" and reference the gating ADR (only ADR-0001, ADR-0003, ADR-0005, ADR-0006, ADR-0012 license edits in Phase 2; ADR-0008 licenses the parity-lint script).
- **`additionalProperties: false` + `schema_version: "v1"` discipline:** every Phase-2 sub-schema (S3-01, S3-02, S4-01, S4-02, S4-03, S5-01, S5-02, S5-03, S5-04, S6-01, S6-02, S7-02, S7-03, S7-04 ×3, S7-05, S7-06 ×5, S7-07 ×2, S7-08 ×2, S7-09, S7-10 ×4) ships with `additionalProperties: false` at every nesting level + `schema_version: "v1"` + an "extra-field rejection" unit test asserting the failing JSON Pointer.
- **Three-layer gitleaks secret defense:** every story touching the gitleaks path must preserve all three layers — wrapper `--redact` (S1-06), Pass 4 sanitizer (S1-09), `x-secret-finding: true` schema tag (S7-03). The adversarial test (`test_gitleaks_redaction_invariant.py`) is the end-to-end pin.
- **Wrapper-enforced `--ignore-scripts`:** every story touching the package-manager wrappers (S1-05's `pnpm`/`npm`-adjacent paths if any; the explicit `BuildGraphProbe` consumer in S3-02) preserves the wrapper-level invariant; the unit-level test `test_pnpm_invariant.py` (S3-02) is the early-surfacing canary.
- **Per-probe local coverage report:** stories S3-01, S3-02, S4-01, S4-02, S4-03, S5-01, S5-02, S5-03, S5-04, S6-01, S6-02, S7-02, S7-03, S7-04, S7-05, S7-06, S7-07, S7-08, S7-09, S7-10 each report local coverage for the new probe in their PR body — the S8-06 ratchet cannot recover if any of these came in under floor.
- **`.codegenie/index/` vs `.codegenie/cache/` lifecycle distinction:** the per-repo SCIP binary is rewritten in place by `SCIPIndexProbe` and **never** auto-deleted; the per-gather caches under `.codegenie/cache/` are LRU-evicted. Stories that touch either namespace must respect the distinction (S4-01, S4-04, S7-01, S7-05).
