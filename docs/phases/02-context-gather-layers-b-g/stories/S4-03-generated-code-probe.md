## Context

`GeneratedCodeProbe` (B4) flags files the Planner must **not** edit: GraphQL-codegen output, OpenAPI/protobuf-generated TypeScript, Prisma client, etc. Editing generated code is silent waste (the next codegen run reverts the change) and a high-confidence false-positive in autonomous-PR pipelines.

The detection strategy is **two-tier, header-first** (per `final-design.md §"Components" §3.4`): (1) cheap header-pattern match — read the first 256 bytes; regex against canonical generator markers (`// @generated`, `/* eslint-disable */ // Generated by`, `# DO NOT EDIT`, `protoc-generated`, etc.); (2) dependency-based detection via Phase 1's `ParsedManifestMemo` — if `@graphql-codegen/cli` / `openapi-typescript` / `prisma` / `protobufjs` appears in `package.json` deps, flag the conventional output directories; (3) **tree-sitter parse only for ambiguous files** that miss the header pattern but live in a suspect directory. The pattern catalog at `src/codegenie/probes/_generated_code_patterns.yaml` is the extension point; new generators are YAML edits, not code edits.

This probe **reuses the per-file findings cache shape declared by S4-02** (`(file_blake3, grammar_version).msgpack` under `.codegenie/cache/tree-sitter/by-file/`) — only the ambiguous-files path writes there. Header-pattern hits are recorded directly in the slice without a per-file cache (the 256-byte read is cheaper than a cache lookup).

## Goal

Ship a deterministic `GeneratedCodeProbe` that flags generated TS/JS files via (a) header-pattern match, (b) dependency-based directory inference from `package.json`, (c) tree-sitter parse fallback for ambiguous files only — driven by the pinned `_generated_code_patterns.yaml` catalog and emitting an `additionalProperties: false` slice that the Planner reads to skip edits.

## Implementation outline

1. **Author `_generated_code_patterns.yaml`.** Three sections per Acceptance criterion 2. Validate each regex is anchored, has bounded backtracking, applies to the first 256 bytes only. Load once at probe instantiation via `safe_yaml.load`, `MappingProxyType`-wrapped, `CatalogLoadError` on malformed.
2. **Sub-schema.** Write `generated_code.schema.json` per Acceptance criterion 3. `additionalProperties: false` everywhere. Cross-link to `SCHEMA-EVOLUTION-POLICY.md`.
3. **Probe implementation.** `src/codegenie/probes/generated_code.py`:
   - **At instantiation**, load and freeze the catalog.
   - **In `run(snapshot, ctx)`:**
     1. **Header pass.** Enumerate files matching `declared_inputs` globs. For each, open in binary mode, read the first 256 bytes, decode as UTF-8 errors=`replace`, match against each `header_patterns[].regex` (anchored, bounded). First hit wins → record `detection: "header", marker: <name>`. No tree-sitter, no cache lookup.
     2. **Dependency pass.** Read `package.json` via `ParsedManifestMemo` (Phase 1). For each `dependency_signals[]` whose `npm_package` appears in `dependencies` or `devDependencies`, glob `output_globs` under `snapshot.root`. For each file not already flagged in step 1, record `detection: "dependency", dependency_signal: <npm_package>`. If `package.json` unavailable → warning, skip the dependency pass.
     3. **Ambiguous-files pass.** For files matching `ambiguous_dirs` paths AND not flagged in steps 1 or 2, compute `file_blake3`; `grammar_version` from `tools.treesitter`; check cache at `.codegenie/cache/tree-sitter/by-file/<blake3>.<grammar_version>.msgpack`. On hit, append findings. On miss, invoke `tools.treesitter.query(file_paths=[file], grammar=<classified>, query_pack=<TS query for synthetic-AST markers>, timeout_s=5.0)`; the query pack here is a *small* set (e.g., "comment node containing /@generated/") — declared inline in the probe module or in a sibling `_generated_code_treesitter.yaml` (acceptable; keep simple). Write atomically to the cache path.
   - **Confidence ladder:**
     - `high` — at least one detection of any kind, no per-file errors.
     - `medium` — `package.json` unavailable (warning) but header path completed.
     - `low` — all three passes errored (rare; degenerate inputs).
4. **Detection priority.** Process header pass before dependency pass; once a file is in `generated_files`, subsequent passes skip it.
5. **Failure handling.** Warning IDs: `generated_code.manifest_unavailable`, `generated_code.pattern_load_failed`, `generated_code.treesitter_timeout`, `generated_code.treesitter_malformed`. Per-file failures isolate.
6. **Atomic cache writes.** Same `tempfile + os.replace` pattern as S4-02. Cache corruption → delete + treat as miss + warning.
7. **Register** in `src/codegenie/probes/__init__.py` (one additive import). Add `generated_code` to envelope `$ref` (optional).

@pytest.mark.asyncio
async def test_header_marker_detected(tmp_path):
    src = tmp_path / "src"; src.mkdir()

## Notes for the implementer

- **Header pattern regexes run on attacker-controllable bytes.** Anchor each (`^`), bound repetition (`{1,100}` not `*`), no nested groups. Pre-compile at module import. If you're tempted to write `.*` — stop. The first 256 bytes is the bound; the regex must be bounded inside that.
- **Detection priority is documented behavior, not an implementation accident.** Header beats dependency beats tree-sitter because header is the most explicit signal (the author *wrote* `@generated`); dependency is inferred; tree-sitter is the catch-all. Encode this with deterministic iteration order; never depend on dict-insertion order accidentally.
- **The pattern catalog is the extension point.** Adding `@auto-generated by ts-rest` is a YAML edit + a test fixture; never a `generated_code.py` edit. If a reviewer asks for "another generator" — it's a one-line YAML patch.
- **Per-file cache reuse from S4-02 is intentional.** Both probes read the same `(file_blake3, grammar_version).msgpack` shape; *different cache entries* because the query packs differ. The shared shape means S7-01's eviction LRU treats both probes uniformly — one access-time clock for all tree-sitter findings.
- **`ParsedManifestMemo` is the load-bearing Phase-1 reuse.** Don't open `package.json` directly; the memo invariant is "parse once per gather". A grep test (`test_no_direct_manifest_open`) would be a good follow-up if Phase 2 starts seeing manifest-parse drift, but isn't load-bearing for this story.
- **`ambiguous_dirs` should be conservative.** False positives (flagging a hand-written file as generated) are higher-cost than false negatives (missing a generated file) — the former silently blocks legitimate edits; the latter only fails to suppress. Err toward narrow `ambiguous_dirs` entries (`src/generated/`, not `src/utils/`).
- **Tree-sitter is in-process** per ADR-0003 / S1-07; no sandbox routing.
- **The slice carries no raw content.** Paths + symbolic detection enum + marker/signal names only. Sanitizer Pass 4/5 idempotence holds trivially.
- A grep for `subprocess`, `import requests`, `import httpx` in `generated_code.py` should return empty.
