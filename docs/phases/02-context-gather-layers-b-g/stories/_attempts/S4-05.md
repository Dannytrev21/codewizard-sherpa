### Diagnosis / lessons (added to `_lessons.md` if generalizable)

- **Validator override #5 was load-bearing.** The original story prescribed reading `ctx.sibling_slices["build_system"]["package_manager"]`. That attribute does not exist on `ProbeContext` (Phase 0 frozen ABC) AND `NodeBuildSystemProbe` does not emit a sibling JSON sidecar in Phase 2. The validator's instruction to re-detect inline was the only way to ship a working probe; the duplication of `_LOCKFILE_PRECEDENCE` is acknowledged in the module docstring with two named follow-up paths (promote to shared helper OR amend Phase 1 to emit a sidecar) — Rule 7 surfaced, not blended.
- **`tier="task_specific"` not `"base"`.** The story's draft AC-1 specified `tier="base"`, but wave-1 dispatches with an empty `detected_languages` snapshot — a base-tier Node-only probe would either spuriously execute on non-Node repos or have to override `applies()`. The cleaner precedent is `NodeBuildSystemProbe` (`tier="task_specific"` + `_admits_node_project`). Rule 7 conflict surfaced in the class docstring with the resolution rationale.
- **`asyncio.to_thread + wait_for` is necessary for sync strategies.** A naïve `asyncio.wait_for(default_dep_graph_registry.dispatch(...))` would NOT interrupt a sync CPU-bound strategy — the dispatch returns synchronously before `wait_for` ever schedules. T-13 fails loud if anyone tries to simplify this back to a single-call form.
- **Empty graph artifact on no-strategy is a Rule 2 (simplicity for the consumer) win.** Phase 3 plugins read one parse path (`{"schema_version": 1, "ecosystem": "<pm>", "nodes": [], "edges": []}`) instead of branching on file-absent. The producer pays one extra `write_bytes`; every consumer avoids a conditional.

The shipped model keeps `str | None` for forward compatibility with Phase 3 reason discriminators we don't want to enumerate yet. A future story can land the sub-ADR + Literal narrowing as a strict superset — every current GREEN test continues to pass.
3. **S4-07 sub-schema for `dep_graph` slice** will pin the slice-dict shape (`ecosystem`, `nodes_count`, `edges_count`, `graph_path`, `confidence`, `reason`). Until then, the slice-dict shape is encoded only in the probe's `_emit` factory.

### Lesson for future stories (`_lessons.md`-eligible)

- **Always verify `sibling_slices`-style claims against `src/codegenie/probes/base.py` before writing the red test.** S4-04, S4-05, and (per validator) future Layer-B probes will keep tripping on this until a probe-sidecar emit pattern lands. The Phase-2 reality: sibling-slice reading means either re-detecting inline OR reading a previously-written JSON file under `<output_dir>/raw/` — never reaching into another probe's in-memory output.
