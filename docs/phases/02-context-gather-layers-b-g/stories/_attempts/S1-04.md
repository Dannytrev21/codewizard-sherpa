### Issues encountered and fixes

1. **`Result[int, str]` is not subscriptable on Python 3.13.** `Result` is `Annotated[Union[Ok[T], Err[E]], Field(...)]` â€” `Annotated[Union, ...]` is not `Generic`, so `Result[int, str]` raises `TypeError: not a generic class`. **Fix:** dropped the `TypeAdapter(Result[int, str])` form from the round-trip test. Replaced with direct `Ok[int].model_validate_json(...)` / `Err[str].model_validate_json(...)` which exercise the same identity contract without needing the subscript. Nested-Ok test uses `Ok.model_validate_json` and asserts the inner Ok survives as a dict (concrete-type recovery requires the outer caller to know the inner generic, which AC-0d does not require). **Lesson logged to `_lessons.md`.**

2. **`TestsExercising` collides with pytest's `Test*` collector.** Imported into a test module's namespace, the Pydantic `BaseModel` subclass triggers `PytestCollectionWarning: cannot collect test class â€¦ has a __init__ constructor`. Per L5 cross-story lesson, aliased on import â€” but the *alias name* matters too: `TestsExercising as TestsExercisingQuery` still starts with `Test` and warns. **Fix:** aliased as `ExerciseTestsQuery` (non-`Test`-prefixed). **Lesson logged.**

3. **Pydantic v2 `union_tag_invalid` error shape.** Initial `_classify` checked `loc[-1] == "compute"`, but Pydantic 2.13 sets `loc = ('derived_queries', 0)` (the list index, not the discriminator name). The discriminator identity lives in `ctx['discriminator']` as `"'compute'"` (literal with single-quotes). **Fix:** branch on `etype == "union_tag_invalid"` and check `ctx["discriminator"] in {"'compute'", "compute"}`; kept the `literal_error` arm with `loc[-1] == "compute"` as defensive fallback. Docstring pins the public contract. **Lesson logged.**

4. **Mypy `--strict` "TypeVar return without TypeVar in args".** `Ok.unwrap_err` and `Err.unwrap` both always-raise; their declared return type was `E` / `T` (unused TypeVar). **Fix:** changed both to `-> NoReturn`. Semantically correct (these never return); matches the "function exits via exception" contract.

5. **Mypy `Missing keys for TypedDict "ErrorDetails"`.** Loader's `first = ve.errors()[0] if ve.errors() else {}` doesn't pass mypy because `{}` isn't a valid `ErrorDetails`. **Fix:** split the empty case explicitly â€” `detail = str(errors[0]["msg"]) if errors else "invalid"`.
