### Design decisions (refactor-step pattern lens)

- **Frozen dataclass `ProbeRegEntry(cls, heaviness, runs_last, registration_index)`** — the Open/Closed gain over `list[type[Probe]]`. Field names chosen to survive a future `KernelRegistryEntry[K, V]` extract (rule-of-three queued by `src/codegenie/indices/registry.py:26` after S1-10 lands the third precedent). Did NOT introduce the kernel base here — Rule 2 + Notes-for-implementer.
- **Dual-shape module-level `register_probe`** via `cls: type[Probe] | None = None` sentinel + `@overload` pair. Backward-compatible with `@register_probe` (no parens, Phase 0/1) and forward-compatible with `@register_probe(heaviness="heavy", runs_last=True)` (Phase 2).
- **`Registry.decorator(*, heaviness, runs_last)`** — per-instance decorator factory used by tests against ad-hoc `Registry()` instances; keeps tests off `default_registry` monkeypatching for the common case.
- **Sort lives in a private module-level `_sort_entries(entries) -> tuple[ProbeRegEntry, ...]`** — functional core; can be reused by `sorted_for_dispatch` and `sorted_for_task` without `self` capture. The two-pass split (non-last sorted, then runs_last sorted) matches the story's prose 1:1 — grep-able, mutation-resistant to a single-key `(runs_last, heaviness_rank, registration_index)` shortcut that would functionally work but obscure the intent.
- **Coordinator integration via a `runs_last_names: frozenset[str]` keyword arg on `gather()`** — the side channel the story Notes asked for. Three partition lists (`base` no-runs_last, `rest` no-runs_last, `runs_last_probes`); the final `rest = rest_non_runs_last + runs_last_probes` puts runs_last at the strict tail regardless of declared `tier`. This is the AC-13 cross-wave invariant.
- **Coordinator emits `coordinator.dispatch.order` twice** — once per wave with the `probe_order` list. Audit anchor (AC-10). Always emitted (even if a wave is empty), so the event is causally bound to gather start.
- **CLI seam split into two helpers**: `_seam_registry_for_task` (returns instances in dispatch order, AC-6a) + new sibling `_seam_runs_last_names` (returns frozenset of names from registry entries with `runs_last=True`). `_seam_coordinator_gather` plumbs both into the underlying `gather()` kwargs. The seam never reads `runs_last` from the probe instance — preserves the ABC freeze.
- **`for_task` left semantically unchanged** — Phase 0/1's `_filter` cache stays intact; the new `sorted_for_task` is layered alongside it. Two cache surfaces are honest (one for the legacy callers, one for the dispatch-side path).

### Refactor decisions (design-patterns lens)

- **Registry + decorator-data** — `ProbeRegEntry` keeps the registry's record-shape Open/Closed. New scheduling annotation (e.g. `runs_first`, `cooperative_yield`) is an additive kwarg + a sort-key extension, never an edit to consumers.
- **Functional core** — `_sort_entries`, `_filter`, `_HEAVINESS_RANK` are pure module-scope; `Registry` is the imperative shell.
- **No premature kernel extract** — `ProbeRegEntry` lives in `probes/registry.py` (not `kernel/registry.py`). The kernel extract is queued for S1-10 (third precedent rule-of-three).
- **Side-channel metadata, not contract pollution** — `runs_last_names: frozenset[str]` keyword on `gather()` is the load-bearing seam between registry data and coordinator behavior. The `Probe` ABC stays frozen.
- **Newtype-shape consistency** — `Heaviness = Literal["light","medium","heavy"]` is the typed discriminator (production ADR-0033 alignment). `_HEAVINESS_RANK`'s key set is pinned to `typing.get_args(Heaviness)` so a future 4th tier surfaces loud at CI.
