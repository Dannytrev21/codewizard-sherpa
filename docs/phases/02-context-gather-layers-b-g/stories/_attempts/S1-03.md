### What broke and how it was fixed (lessons → `_lessons.md`)

1. **`@runtime_checkable` `isinstance(inst, proto)` lazily creates `__annotations__` on the inspected class.** The dynamic walk in AC-6 iterated every importable class under `codegenie` and ran `isinstance` against the four adapter protocols. As a side effect, marker exceptions (e.g., `MalformedJSONError`) gained an empty `__annotations__` entry in `__dict__`, tripping `tests/unit/test_errors.py::test_subclasses_are_markers_only` (which asserts marker classes hold only a small allowed-key set). Fix: short-circuit the walk on `issubclass(cls, BaseException)` — exception classes cannot satisfy any adapter protocol anyway. → see **L4** in `_lessons.md`.

2. **`TestInventoryAdapter` triggers `PytestCollectionWarning` because the name starts with `Test`.** Pytest tries to collect any `Test*` class in a test-file namespace as a test class. A `__test__ = False` *inside* the Protocol body adds it to the Protocol's required-attribute set (PEP 544 attribute presence check), so stubs would have to declare `__test__` too. Fix: set `TestInventoryAdapter.__test__ = False` **after** the class body — pytest reads the attribute, Protocol's `_get_protocol_attrs` does not. → see **L5** in `_lessons.md`.

3. **Ruff `UP007` rewrites `Union[X, Y, Z]` → `X | Y | Z`** even inside `Annotated[..., Field(discriminator=...)]`. Pydantic v2 handles both forms; the rewrite is safe. Pre-existing **L2** lesson; no change needed.

### Refactor decisions

- Did **not** extract a shared `HasConfidence(Protocol)` base — story Notes-for-implementer §"Deliberate non-extraction" explicitly forbids it: four parallel one-line declarations are not duplication; a shared base would smuggle in the Abstract-Factory shape `phase-arch-design.md §Design patterns` rejects.
- Adopted `slots=True` on `Occurrence` per story Notes §"`Occurrence` and `slots=True`": Phase 3's `ScipAdapter` will mmap-walk SCIP blobs and construct millions of instances; `__slots__` cuts per-instance memory ~30%. Added a dedicated test (`test_occurrence_uses_slots`) asserting `not hasattr(inst, "__dict__")`.
- Set `TestInventoryAdapter.__test__ = False` after the class body to silence pytest's collection warning without polluting the Protocol attribute set.
