### Trip-ups encountered (lessons for next story)

1. **AC-6 source scan false-positive on errors.py docstring.** The original
   `DepGraphRegistryError` docstring contained the literal token
   `@register_dep_graph_strategy`; the AC-6 source scan flagged it as a
   Phase-2 strategy registration. **Fix:** rephrased the docstring to omit
   the decorator-application token (`@` + name) and added an explicit note
   about why. The fix preserves the AC's strict-substring scan (no
   weakening to "in `src/codegenie/depgraph/`").

2. **`ruff` `B017` bans `pytest.raises(Exception)`.** Replaced with the
   specific `pydantic.ValidationError` exception class (frozen-assignment
   and `extra=forbid` both raise it under Pydantic v2). The story TDD plan
   suggested `pytest.raises(Exception)` for the frozen test — `ruff` is
   stricter; the codebase convention wins (Rule 11).

3. **`ruff` `UP037` bans `"networkx.DiGraph"` quoted return annotation when
   `from __future__ import annotations` is in effect.** Removed the quotes
   on the `dispatch` return type — the forward reference works without
   them because PEP 563 defers evaluation. Kept the `TYPE_CHECKING` import
   guard so the registry module doesn't pay the `networkx` import cost at
   `codegenie.depgraph.registry` load time (cold-start defense).

4. **`networkx` has no `py.typed` marker.** Added an `ignore_missing_imports`
   override scoped to `networkx` and `networkx.*` (parallel to the existing
   `pyarn` override). Pinned the rationale in a comment so a future story
   that adds typed `networkx` usage knows to revisit.

5. **mypy `redundant-cast` on `cast(PackageManager, "pnpm")`.** Removed the
   casts; bare string literals narrow directly to `Literal["pnpm", …]`
   when annotated as `PackageManager`. The story TDD plan's `cast(...)`
   pattern was defensive against a hypothetical wider type — under
   `mypy --strict` the narrow-from-literal is unambiguous.

6. **List invariance under `--strict`.** Strategy functions in the tests
   were typed as `list[dict[str, object]]` but `DepGraphStrategy` expects
   `list[Mapping[str, Any]]`. `list` is invariant; the test signatures
   had to match exactly. **Lesson:** when copying signatures from a TDD
   plan, prefer the exact alias' inner type (`list[Mapping[str, Any]]`)
   over the structural-superset (`list[dict[str, object]]`).

7. **`test_packaging.RUNTIME_DEPS` pins the runtime closure.** Adding any
   new `[project].dependencies` entry breaks
   `test_runtime_dependencies_are_exactly_adr_0006_closure` until the
   frozenset literal is widened. Added `"networkx"` with a rationale
   comment about the ADR-0002 fence still excluding LLM SDKs.

### Refactor decisions

- **Rule-of-three kernel-extract opportunity** for a shared `KernelRegistry[K, V]`
  base (across `probes/registry.py`, `indices/registry.py`,
  `depgraph/registry.py`) — **deferred**, per story §Notes for the
  implementer. The three sites' `dispatch` shapes diverge non-trivially
  (`for_task` filter + LRU / `dispatch_all` total / single-`dispatch` +
  `has_strategy` query); the cost of introducing a generic mid-phase
  exceeds the duplication cost. Mirrors S1-02's own deferral.
- **Module docstring** cites 02-ADR-0006 §Decisions noted (registry
  symmetry), production ADR-0033 (typed identifiers), arch §"Component
  design" #11, and the Phase-3 hand-off path
  (`plugins/vulnerability-remediation--node--npm/strategies/dep_graph_pnpm.py`).
- **`unregister_for_tests`** kept deliberately-awkward, parallel to S1-02.

### Cross-story lessons to append

- **L17:** Phase-2 stories adding a new runtime dependency must update both
  (a) `pyproject.toml` `[project].dependencies` AND (b)
  `tests/unit/test_packaging.py` `RUNTIME_DEPS` AND (c) potentially add a
  `[[tool.mypy.overrides]]` entry if the package lacks `py.typed`. The
  full triple lands in one commit alongside `uv lock` regeneration.
- **L18:** Marker docstrings under `codegenie.errors` must NOT contain the
  literal decorator-application token (`@register_foo`) of any registry
  guarded by a Phase-2 zero-registration AC. The substring scan is strict
  on purpose; phrase the docstring around the function name without `@`.
- **L19:** Story TDD plans that suggest `cast(<Literal alias>, "value")`
  trip mypy `redundant-cast` once the LHS is annotated. Prefer
  `NAME: <Literal alias> = "value"` — narrows directly, no cast.
