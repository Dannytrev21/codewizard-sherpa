### Diagnosis

Two cycles internally before the green snapshot above:

1. **Import cycle** between `codegenie.indices.registry` → `codegenie.types.identifiers` → `codegenie.probes.node_build_system` → `codegenie.probes.__init__` → `codegenie.probes.layer_b.index_health` → `codegenie.indices.registry` (mid-load). Fix: moved the `IndexName` import in `indices/registry.py` to a `TYPE_CHECKING` block. Safe because `NewType` is identity-to-`str` at runtime and `from __future__ import annotations` keeps every annotation site as a string.
2. **`mypy --warn-unreachable` complained** about the single-level `match` in `_derive_confidence` because mypy cannot narrow `Stale.reason: StaleReason` after the four `Stale(reason=Variant())` cases. Fix: nested `match` — outer on `freshness: IndexFreshness` (Fresh / Stale), inner on `reason: StaleReason` (CoverageGap / CommitsBehind / DigestMismatch / IndexerError). Both arms now end in `assert_never` — DP3 (producer/consumer `assert_never` ladder) is structurally enforced at both nesting levels.

Also a minor cycle: T-15b initially asserted "exactly one **Await** of `dispatch_all`" — but the registry's `dispatch_all` is synchronous (per S1-02). Pivoted to "exactly one **Call** of `dispatch_all`" — same Open/Closed enforcement spirit (no per-index branching detectable in the AST), zero monkey-patching needed.

### Refactor decisions (DP1-DP4 lens)

- **DP1 — Open/Closed at the file boundary.** `run()` contains zero `if index_name == "..."` branches; every per-index decision is inside a registered check function in its owning module. The two complementary AST tests (T-15a negative + T-15b positive) co-witness the discipline. **Applied.**
- **DP2 — Functional core / imperative shell.** `read_raw_slices`, `_derive_confidence`, `_last_indexed_at`, `_demote_min` are all pure module-level helpers. `IndexHealthProbe.run` is the imperative shell that composes them + the registry call + two `git` calls. No hidden state, no class fields beyond Probe contract. **Applied.**
- **DP3 — Producer/consumer `assert_never` ladder.** B2 produces `IndexFreshness`; S8-01's `confidence_section` (lands later) consumes. Both ends `match` exhaustively; `mypy --warn-unreachable` enforces. **Applied.** Documented in the module docstring (top of `index_health.py`).
- **DP4 — `IndexName` newtype.** Used wherever the typed identifier is meaningful. Did NOT pre-extract a shared `KernelRegistry[K, V]` despite rule-of-three reach — Rule 2 (simplicity first); the fourth registry arrival is the right trigger. **Deferred.**

### Cross-story lessons

- Wherever a Phase-2+ module imports from `codegenie.types.identifiers`, prefer `TYPE_CHECKING` for `NewType`-derived names. The runtime identity-to-`str` semantics mean nothing breaks; the cycle-break is free.
- AC-3b's "B2 starts after every sibling ends" is only a temporal invariant under single-slot semaphore. Under default concurrency, runs_last is *dispatch-position* only. Future runs_last assertions should pin which mechanism they're testing.
- The forbidden-patterns hook's per-path scoping (via `applies_when`) is the right tool for "this rule applies only to one module" — adding a fifth or sixth such rule is just one `Rule(...)` entry, zero edits to `_scan_file()` / `main()`. Open/Closed at the script boundary.
