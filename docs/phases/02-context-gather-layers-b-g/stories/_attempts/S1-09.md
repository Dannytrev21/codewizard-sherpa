### Design decisions (refactor-step pattern lens)

- **Tuple rename, not registry-extract.** The story's Notes-for-implementer flag the rule-of-three queue: this is the second `ProbeContext` callable amendment (ADR-0002 + ADR-0004). When a Phase 3+ ADR adds a third optional callable, the right shape becomes a `_ALLOWED_PROBE_CONTEXT_FIELDS_BY_ADR: Mapping[ADRId, frozenset[str]]` registry (Open/Closed at the file boundary). Per Rule 2, kept the tuple flat with both ADRs cited in the failure message; left a single drift-message constant (`_FIELD_LIST_DRIFT_MESSAGE`) used by both `test_probe_context_field_list_matches_allowed` and the two synthetic mutation killers so the message stays in one place.
- **Mutation-killer pattern reused, not invented.** Mirrored `test_probe_context_sentinel_fires_on_synthetic_third_field` for the ADR-0004 case: synthetic 9-field dataclass via `@dataclasses.dataclass`, `pytest.raises(AssertionError, match=r"ADR-0004")`. No subprocess, no flakiness — matches the codebase idiom per Rule 11.
- **Annotation pin via `inspect.get_annotations` + `repr()` substring assertions.** Catches all three retypes: drop `| None` return, drop `Path` arg, widen to `dict[Path, str]`. Same shape as the Phase-1 `test_probe_context_new_field_annotations_pinned` test.
- **Doc-grep test added** (`test_localv2_section_4_shows_image_digest_resolver`) — names *what* drifted when Tier 1 doc_fingerprint test fires; without it the failure-message is an opaque hash mismatch.
- **Idempotence test via subprocess invocation** of the regen script — runs twice and asserts byte-equality, plus `before == after_second` to catch a stale-committed snapshot. Mirrors the S7-03 golden-stability discipline.
- **Existing `test_probe_context_sentinel_fires_on_synthetic_third_field` kept and updated.** The synthetic 8-field dataclass it builds (with a future `_third_field` addition past `parsed_manifest`/`input_snapshot`) is still a valid Phase-1 violation case; updated its regex to `r"ADR-0002|ADR-0004"` and routed through the shared `_FIELD_LIST_DRIFT_MESSAGE` since the sentinel tuple now spans both ADRs.
