### Refactor decisions

- **`Rule` dataclass IS the load-bearing design change.** Three rule
  kinds (literal-pattern regex, anchored regex with negative lookbehinds,
  path-scoped regex) crosses the rule-of-three threshold. Rule of three
  is satisfied AT the third site (story §Notes for the implementer), so
  the abstraction lands now — not premature, not deferred. Adding a
  Phase-3 rule (e.g., ban `httpx` under `src/codegenie/plugins/`) is one
  new `Rule(...)` entry — zero edits to `_scan_file()` / `main()`. AC-15
  is the structural pin.
- **`applies_when` predicate, not a `match`-on-rule-kind.** A tagged-
  union with `LiteralRule | AnchoredRule | PathScopedRule` was
  considered but rejected — every variant has the same evaluation shape
  (regex against text, optionally gated on path), so a single dataclass
  with one optional predicate is the simpler primitive. The Path
  predicate is open: any future scoping dimension (file size, age,
  content prefix) collapses to a `Callable[[Path], bool]` swap.
- **`pathlib.Path.parts` lookup, not a regex on the path string.** The
  story §Notes called this out; verified the helper is symlink/Windows
  safe. Never regex against the OS-formatted path for a structural check
  — `parts[idx + 1] in _PHASE2_BANNED_PACKAGES` is the OS-neutral form.
- **`slots=True` on `Rule`.** Kept; the test bent (registered module in
  `sys.modules`) instead of weakening the production dataclass shape.
  See L25 for the loader-vs-slots interaction.
- **NO edits to `pyproject.toml`.** Verified at validation time —
  `warn_unreachable = true` is already repo-wide from Phase 0 S1-02
  (commit `3944f02`). Adding `[[tool.mypy.overrides]]` blocks would be
  redundant and would expand the config surface unnecessarily. The
  arch's per-module prescription is honored broader-than-arch; the
  narrowing decision is an S8-04 backlog item per story §Validation
  notes #1.
- **NO `tests/conftest.py` collect_ignore.** The fixture file is named
  `incomplete_match.py` (not `test_*.py`); pytest's default discovery
  skips it. The story §Notes hedged on this; the simpler shape held.

### Cross-story lessons to append

- **L25:** `dataclass(slots=True)` under
  `from __future__ import annotations` resolves string-form annotations
  via `sys.modules[cls.__module__].__dict__`. When loading a module via
  `importlib.util.spec_from_file_location` + `exec_module` (e.g., to
  introspect a `scripts/` file that isn't a package), the loader MUST
  register the module in `sys.modules` BEFORE `exec_module` — otherwise
  slot synthesis raises `AttributeError: 'NoneType' object has no
  attribute '__dict__'`. Standard library docs cover this under the
  "Approximating importlib.import_module" recipe but it's not surfaced
  in dataclass docs. Any future Phase-2 test that introspects a
  `scripts/` module hits the same trap.
- **L26:** Phase 2 stories that extend `scripts/check_forbidden_patterns.py`
  must scope rules via the `applies_when` predicate, NOT via
  `.pre-commit-config.yaml`'s `files:`/`exclude:` regex. The reason: the
  test surface (subprocess invocation in `tmp_path`) and the runtime
  surface (pre-commit invocation on staged files) MUST be the same; YAML
  scoping bypasses the test surface and produces silent rule-coverage
  gaps. AC-1 prose pins this for S1-11; future Phase-3 path-scoped rules
  (e.g., ban `httpx` under `plugins/`) inherit the discipline.
