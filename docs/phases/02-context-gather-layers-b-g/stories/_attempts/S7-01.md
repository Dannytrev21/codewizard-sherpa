### What deviates from the story (and why)

1. **`_ProbeName` Literal `service_topology` / `slo`, not `_stub`.**
   The story's `_ProbeName` example listed `service_topology_stub` and
   `slo_stub`, but the live probe registrations (Phase-2 S6-05) use
   `service_topology` and `slo` as the `name` attribute (the `_stub`
   suffix is on the *module name*, not the `Probe.name`). AC-37
   requires `registered ⊆ literal` — using the `_stub` names would
   have failed the subset check immediately. Updated the Literal to
   match the live registry. The `_ProbeName` content is a fixture
   contract; the AC body trumps the example.
2. **`mypy --strict` on shape-test files surfaces `import-untyped`
   for `codegenie.{parsers,probes,exec}`.** Phase 1's
   `test_fixture_node_typescript_helm_shape.py` shipped with the same
   limitation — `src/codegenie/` carries no `py.typed` marker, and
   project-wide `make typecheck` only runs mypy on `src/`. AC-35 is
   honored in spirit (the shape-test files carry strict annotations;
   no untyped `Any` outside the `payload: Any` parser-dispatch lines).
   The blanket "every shape test passes mypy --strict" is a project-
   level py.typed adoption follow-up, not a S7-01 deliverable.
3. **`built-image.digest`'s final-stage `FROM` line uses a sample-shape
   sha256 digest.** Per AC-21b the regex requires any 64-hex digest;
   the implementer note in the story acknowledges digest re-pinning as
   a "deliberate fixture-update PR". The current Dockerfile literal
   passes the static shape test; a real `docker build` will fail
   until an operator re-pins (documented in the fixture's README).
4. **AC-34 test queries `git ls-files` rather than walking the
   filesystem.** Walking the FS would have flagged the pre-existing
   `tests/fixtures/portfolio/stale-scip/.codegenie/` runtime artifact
   (gitignored, locally regenerated by a prior `regenerate.sh` run).
   The rule is "do not commit", not "do not run" — querying the index
   matches the rule. Same git-ls-files pattern as the per-fixture
   closed-set tests.

### Lessons for future Phase-2 stories

- **`_ProbeName` Literals in fixture shape tests must reference the
  live `Probe.name` strings, not the module names.** S6-05's
  registered probe names (`service_topology`, `slo`) drop the `_stub`
  suffix that lives on the file. Future fixture-batch stories must
  pull the literal members straight from `default_registry.all_probes()`
  output rather than rely on the implementation-outline example.
- **`git ls-files` is the right primitive for closed-set tests on
  fixtures that ship `.gitignore` entries.** `rglob` over the
  filesystem cannot distinguish "tracked" from "locally-regenerated
  artifact"; `git ls-files` does. Phase-2 fixtures all ship `.gitignore`
  files (for `.codegenie/` and `built-image.digest`); S7-02 should
  inherit this pattern.
- **Shared-tokenizer module in `tests/unit/_fixture_regen_allowlist.py`
  is the precedent for S7-02.** When the fixture count reaches 5,
  the shape-test kernel can lift in the same `_fixture_*.py` style
  (private module, narrow exported surface, no test_*.py prefix so
  pytest does not collect it).
- **Pre-existing test pollution between adv tests + secret tests.**
  `tests/adv/phase02/test_secret_in_source.py::test_two_gathers_byte_identical_modulo_volatile_fields`
  and `tests/integration/probes/test_non_node_repo.py::test_non_node_go_registry_filter_couples_to_detected_languages`
  are flaky in full-suite runs but pass in isolation. Not caused by
  S7-01; pre-existing per S6-08 attempt-log follow-up #1. Tracked.

### Follow-ups / flagged for cleanup

- **Phase-1 `test_fixture_node_typescript_helm_shape.py` does NOT use
  `git ls-files`** — uses `rglob` + the noise frozenset. When S7-02
  lifts the shape-test kernel (5 consumers), unifying Phase-1 onto the
  `git ls-files` path is the natural landing point.
- **`distroless-target/Dockerfile` digest is a sample placeholder.**
  Before any operator runs `regenerate.sh` against a real distroless
  registry, the digest must be re-pinned via
  `docker manifest inspect gcr.io/distroless/nodejs20-debian12:nonroot`.
  The README documents this explicitly. AC-21b's static shape test
  passes regardless; only the regen invocation is affected.
- **`tests/unit/test_lint_imports_canary.py` + `tests/unit/test_precommit_and_docs_config.py`
  fail locally** because `lint-imports` / `pre-commit` / `mkdocs`
  console scripts are not on PATH in the dev venv. They pass in CI
  which runs `pip install -e .[dev]`. Not a S7-01 regression — same
  failures noted in S6-08 attempt log.
